# 学习包开发规范（ManYu）

> 本文档基于 foundation-1~3 三个已发布的包总结提炼，供后续所有学习包（foundation-4~10、course-\*、daily-\*、exam-\*）开发时参考。
>
> **最后更新**：2026-07-03

---

## 一、概述

### 1.1 包类型

| 类型 | 前缀 | 说明 | 示例 |
|------|------|------|------|
| **基础包** | `foundation-` | 按语法递进组织，覆盖完整英语基础 | `foundation-1-beginner` |
| **日课包** | `daily-` | 按生活场景组织，侧重日常实用 | `daily-campus`, `daily-travel` |
| **语法课程包** | `course-` | 按语法专题组织，深度练习 | `course-tenses`, `course-modal-verbs` |
| **考试包** | `exam-` | 按考试大纲组织，备考训练 | `exam-cet-4-life`, `exam-ielts-6` |

### 1.2 目录结构

每个包是一个独立目录，放在 `apps/backend/prisma/data/packages/<package-name>/` 下，包含以下文件（带 ✅ 为必需，带 ⭐ 为强烈推荐，其余为可选）：

```
packages/<package-name>/
├── 学习包的功能介绍.md            # ⭐ 功能文档（面向学习者和开发者）
├── scenes.csv                    # ✅ 场景定义
├── training_topics.csv           # ✅ 训练话题
├── chunks.csv                    # ✅ 句块
├── scene_vocabulary.csv          # ✅ 词汇
├── sentence_patterns.csv         # ✅ 句型
├── script_episodes.csv           # ✅ 剧本关卡
├── episode_chunks.csv            # ✅ 关卡↔句块关联
├── warmup_pipeline.json          # ⭐ 热身输出流水线
└── ink-scripts/                  # ⭐ Ink 互动剧本
    ├── practice_<key>.ink
    └── ...
```

## 二、CSV 文件规范

### 2.1 通用规则

- **编码**：UTF-8 without BOM
- **分隔符**：逗号 `,`。若字段内容含逗号，用双引号包裹
- **空值**：留空即可，不要写 `null` / `NULL`
- **列名**：snake_case，小写
- **首列**：不需要行号或 ID，主键由 Prisma 自动生成

### 2.2 scenes.csv — 场景

定义本包覆盖的真实生活/学习场景。

**列定义：**

| 列 | 类型 | 必填 | 说明 |
|:---|:-----|:----:|:-----|
| `category_name` | string | ✅ | 场景分类名，如 `自我介绍`、`日常习惯`。同分类的场景会归组 |
| `title` | string | ✅ | 场景标题，如 `我是谁`、`起床与出门`。**唯一**（跨包不要求唯一，但同一包内必须唯一） |
| `location` | string | — | 场景发生地点描述，如 `at a party`、`in the morning` |
| `required_output_level` | integer | — | 所需输出等级（1=L1, 2=L2, 3=L3）。留空则不限 |
| `required_user_level` | integer | — | 所需用户等级。留空则不限 |
| `description` | string | — | 场景简要描述 |
| `package_type` | string | — | 包类型标记。可选值：`daily`、`exam`、`story`、`course`、`foundation` |

**示例：**

```csv
category_name,title,location,required_output_level,required_user_level,description,package_type
日常寒暄,打招呼告别,at a party,1,,见面问好和离开道别,
日常寒暄,简单闲聊,on campus,1,,聊天气和近况,
```

### 2.3 training_topics.csv — 训练话题

每个话题对应一个输出训练环节，也是 `warmup_pipeline.json` 的顶层 key。

**列定义：**

| 列 | 类型 | 必填 | 说明 |
|:---|:-----|:----:|:-----|
| `scene_title` | string | ✅ | 所属场景的 title（对应 `scenes.csv` 的 `title` 列） |
| `title` | string | ✅ | 话题标题，如 `打招呼与告别`、`我是谁`。**包内唯一** |
| `prompt_en` | string | ✅ | 英文提示语，在 AI 对话中用作 system prompt |
| `prompt_zh` | string | ✅ | 中文提示语 |
| `duration_sec` | integer | — | 建议练习时长（秒），如 `300` |
| `difficulty` | integer | — | 难度等级（1~5） |
| `description` | string | — | 话题简要说明 |
| `knowledge_points` | string | — | 知识点标签，逗号分隔，如 `be动词, 人称代词` |
| `teaching_markdown` | text | — | 教学指导（Markdown 格式），在练习前显示给学习者 |
| `ink_script_key` | string | — | 关联 Ink 互动剧本的 key（对应 .ink 文件 YAML front matter 中的 `key`） |

**示例：**

```csv
scene_title,title,prompt_en,prompt_zh,duration_sec,difficulty,description,knowledge_points,teaching_markdown,ink_script_key
我是谁,我是谁,Practice introducing yourself. Say your name, age, and where you are from.,练习自我介绍。说出你的名字、年龄和来自哪里。,300,1,最基本的自我介绍,be动词, 人称代词,我是...,self-intro
我是谁,年龄与数字,Ask and answer about age. Learn numbers 1-100.,询问和回答年龄。学习数字1-100。,300,1,数字与年龄表达,数字, 年龄, `How old`,age-and-numbers
```

### 2.4 chunks.csv — 句块

句块是"可直接背的完整句子"，不是碎片单词。

**列定义：**

| 列 | 类型 | 必填 | 说明 |
|:---|:-----|:----:|:-----|
| `scene_title` | string | ✅ | 所属场景 title |
| `topic_title` | string | — | 如果指定，该句块只关联到此话题；留空则关联到场景下所有话题 |
| `category` | string | — | 分类标签，如 `自我介绍`、`日常动作` |
| `text` | string | ✅ | 英文句块，**完整自然句子**，如 `My name is Li Ming.` |
| `meaning` | string | ✅ | 中文释义 |
| `difficulty` | integer | — | 难度（1~5） |
| `description` | string | — | 补充说明 |
| `examples_json` | text | — | 替换练习示例，JSON 数组格式：`[{"en": "替换后句子", "zh": "中文", "note": "说明", "level": 1}]` |

**设计原则：**

```
✅ 正确： "I am from China."
✅ 正确： "She gets up at 7 o'clock."
❌ 错误： "be from"（短语而非句子）
❌ 错误： "get up"（短语而非句子）
```

**示例：**

```csv
scene_title,topic_title,category,text,meaning,difficulty,description,examples_json
我是谁,我是谁,自我介绍,"My name is Li Ming.","我叫李明。",1,,,[{"en":"My name is Tom.","zh":"我叫汤姆。","note":"替换名字"}]
我是谁,我是谁,自我介绍,"I am from China.","我来自中国。",1,,,
```

## 三、Warmup Pipeline 规范

### 3.1 顶层结构

```json
{
  "话题标题": {
    "outputTraining": {
      "version": 1,
      "enabled": true,
      "pipeline": [
        // 练习步骤数组（至少 3 个步骤）
      ]
    }
  }
}
```

- 顶层 key 必须与 `training_topics.csv` 中的 `title` 完全一致
- 每个话题至少包含 **3 个练习步骤**，推荐 3~5 个
- `version` 固定为 `1`

### 3.2 练习类型总览

| 类型（`type`） | 常用 `id` | 方向 | 最少条目数 | 说明 |
|:---------------|:----------|:-----|:----------:|:-----|
| `chunk_substitution` | `cs_zh_0` | `zh_to_en` | 3 | 中译英替换练习，核心练习 |
| `chunk_substitution` | `cs_en_1` | `en_to_zh` | 2 | 英译中替换练习，理解检查 |
| `chunk_substitution` | `fl_3` | `zh_to_en` | 3 | 末尾词（句子尾词替换变体） |
| `pattern_drill` | `pd_1` / `pd_2` | `zh_to_en` | 2 | 句型操练，固定模板换槽位 |
| `vocab_sentence_building` | `vsb_1` | `zh_to_en` | 2 (patterns) | 一词多句，同一词在不同句型 |
| `sentence_decomposition` | `sd_3` | — | 2 (levels) | 逐步构建，从简到繁 |

### 3.3 chunk_substitution — 替换练习

**中译英（zh_to_en）：**

```json
{
  "id": "cs_zh_0",
  "type": "chunk_substitution",
  "kind": "word",
  "direction": "zh_to_en",
  "title": "我是谁 · 中译英",
  "chunk": "name",
  "chunkMeaning": "",
  "items": [
    {
      "zh": "你好！我是李明。",
      "answer": "Hello! I am Li Ming.",
      "targetWord": "name"
    },
    {
      "zh": "她叫小红。",
      "answer": "Her name is Xiao Hong.",
      "targetWord": "name"
    },
    {
      "zh": "你叫什么名字？",
      "answer": "What is your name?",
      "targetWord": "name"
    }
  ]
}
```

**英译中（en_to_zh）：**

```json
{
  "id": "cs_en_1",
  "type": "chunk_substitution",
  "kind": "word",
  "direction": "en_to_zh",
  "title": "我是谁 · 英译中",
  "chunk": "name",
  "chunkMeaning": "",
  "items": [
    {
      "en": "My name is Li Ming.",
      "answer": "我叫李明。",
      "targetWord": "name"
    },
    {
      "en": "What is your name?",
      "answer": "你叫什么名字？",
      "targetWord": "name"
    }
  ]
}
```

**规则：**
- `chunk` 字段填写**目标词/短语**（或留空）
- `items` 至少 **3 条**（cs_zh_0）、**2 条**（cs_en_1）
- `targetWord` 必须存在于 `scene_vocabulary.csv` 中
- `answer` 是**完整正确的翻译句子**
- 同一话题内，cs_zh_0 和 cs_en_1 的内容可以有交叉但不应完全重复

### 3.4 chunk_substitution fl_3 — 末尾词（变体）

出现在 `foundation-2-daily-life`，用于句子末尾词替换练习：

```json
{
  "id": "fl_3",
  "type": "chunk_substitution",
  "kind": "word",
  "direction": "zh_to_en",
  "title": "几点起床 · 末尾词",
  "chunk": "",
  "chunkMeaning": "",
  "items": [
    {
      "zh": "我7点起床。",
      "answer": "I get up at 7.",
      "targetWord": "get up"
    },
    {
      "zh": "每天早上7点起床。",
      "answer": "Get up at 7 every morning.",
      "targetWord": "wake up"
    },
    {
      "zh": "他6点起床。",
      "answer": "He gets up at 6.",
      "targetWord": "get up"
    }
  ]
}
```

**适用场景：** 当话题的核心练习需要强调句末关键词时使用。

### 3.5 pattern_drill — 句型操练

```json
{
  "id": "pd_2",
  "type": "pattern_drill",
  "title": "我是谁 · 句型",
  "pattern": "I am ___.",
  "patternMeaning": "我是...",
  "slots": [
    "Li Ming",
    "a student",
    "Tom"
  ],
  "direction": "zh_to_en",
  "items": [
    {
      "zh": "我是...（名字）",
      "answer": "I am Li Ming.",
      "pattern": "I am ___.",
      "slotValues": {
        "slot1": "Li Ming"
      }
    },
    {
      "zh": "我是...（身份）",
      "answer": "I am a student.",
      "pattern": "I am ___.",
      "slotValues": {
        "slot1": "a student"
      }
    }
  ]
}
```

**规则：**
- `pattern` 中的 `___` 代表槽位
- `slots` 数组给出所有可选槽位值
- `items` 至少 **2 条**，每条使用 `slotValues` 指定填充的值
- `slot1` 对应第一个 `___`，`slot2` 对应第二个，以此类推
- `pattern` 字段在 items 中可以重申（当模板一致时）或不同（当需要变换模板时）
- `patternMeaning` 给出模板的中文含义

**多槽位示例（pd_2 变体）：**

```json
{
  "id": "pd_2",
  "type": "pattern_drill",
  "title": "场景描述 · 句型",
  "pattern": "There was a ___ in the ___. It was very ___.",
  "patternMeaning": "...里有一个...它很...。",
  "slots": [
    "big dog the park friendly",
    "cat the garden cute"
  ],
  "direction": "zh_to_en",
  "items": [
    {
      "zh": "...里有一个...它很...。",
      "answer": "There was a big dog in the park. It was very friendly.",
      "pattern": "There was a ___ in the ___. It was very ___.",
      "slotValues": {
        "slot1": "big dog the park friendly",
        "slot2": "cat the garden cute"
      }
    }
  ]
}
```

> **注意**：多槽位时，`slots` 和 `slotValues` 中的值用**空格**分隔多个槽位值，需要对应 pattern 中 `___` 的顺序。

### 3.6 vocab_sentence_building — 一词多句

让学习者用一个核心词汇/短语在 2~3 个不同句型中造句：

```json
{
  "id": "vsb_1",
  "type": "vocab_sentence_building",
  "title": "我的信息 · 一词多句",
  "vocabWord": "My name is / I am from / I am a",
  "vocabMeaning": "我叫 / 我来自 / 我是",
  "direction": "zh_to_en",
  "patterns": [
    {
      "chunk": "My name is ___. I am from ___.",
      "items": [
        { "zh": "我叫李明。我来自中国。", "answer": "My name is Li Ming. I am from China." },
        { "zh": "我叫汤姆。我来自美国。", "answer": "My name is Tom. I am from America." }
      ]
    },
    {
      "chunk": "My name is ___. I am a ___.",
      "items": [
        { "zh": "我叫李明。我是一个学生。", "answer": "My name is Li Ming. I am a student." },
        { "zh": "我叫小红。我是一个老师。", "answer": "My name is Xiao Hong. I am a teacher." }
      ]
    }
  ]
}
```

**规则：**
- `vocabWord` 列出本练习涉及的核心词汇/短语
- `patterns` 至少 **2 组**，每组至少 **2 条**
- 每组聚焦一个句型模式
- 适合在 `cs_zh_0` + `cs_en_1` + `pd` 之后的第 4 个步骤使用

### 3.7 sentence_decomposition — 逐步构建

将一句话从简单到复杂逐步构建：

```json
{
  "id": "sd_3",
  "type": "sentence_decomposition",
  "title": "我的信息 · 逐步构建",
  "levels": [
    {
      "level": 1,
      "label": "说名字",
      "en": "My name is Li Ming.",
      "zh": "我的名字是李明。",
      "highlight": "name",
      "hint": "先说名字"
    },
    {
      "level": 2,
      "label": "加来源",
      "en": "My name is Li Ming. I am from China.",
      "zh": "我的名字是李明。我来自中国。",
      "highlight": "from",
      "hint": "加上来自哪里"
    },
    {
      "level": 3,
      "label": "加年龄",
      "en": "My name is Li Ming. I am from China. I am 20 years old.",
      "zh": "我的名字是李明。我来自中国。我20岁。",
      "highlight": "old",
      "hint": "加上年龄"
    }
  ]
}
```

**规则：**
- **每级必须在前一级基础上累加**（不能换场景换主语）
- 至少 **2 级**，推荐 3 级
- `highlight` 字段标注本级别的新增关键词
- `hint` 字段提供学习提示

### 3.8 Pipeline 编排推荐

每个话题的 pipeline 推荐按以下顺序编排：

| 序号 | 类型 | 目的 | 推荐条目数 |
|:----:|:-----|:-----|:----------:|
| 1 | `cs_zh_0`（中译英） | 激活已知内容，从中文出发 | 3~5 |
| 2 | `cs_en_1`（英译中） | 理解检查，确认能听懂 | 2~3 |
| 3 | `pd_1` 或 `pd_2`（句型操练） | 结构输出，强化语法模式 | 2~4 |
| 4（可选） | `vsb_1`（一词多句） | 灵活运用，一词多用 | 2~3 patterns |
| 5（可选） | `sd_3`（逐步构建） | 复杂句子的渐进掌握 | 2~3 levels |

**最少要求：** 每个话题至少 3 个步骤（cs_zh_0 + cs_en_1 + pd）。

> **注意**：新增 `vsb_1` 或 `sd_3` 时，确保它们在 pipeline 数组**内部**，不要放到数组外导致 JSON 结构错误。

## 六、设计原则

### 6.1 场景优先，语法穿插

不按"语法点 → 例句"模式，而是**按真实使用场景组织**。学习者在场景中自然遇到语法点。语法只是辅助标签，不是主线。

### 6.2 句块 = 完整句子

每个句块必须是**可直接用于对话的完整句子**，不是碎片单词或短语。

```
✅ "I am from China."
✅ "She gets up at 7 o'clock."
❌ "be from"（无法直接使用）
❌ "get up"（无法直接使用）
```

### 6.3 递进关系

包与包之间要有明确的递进关系：

```
① be 动词基础
  → ② 一般动词与日常生活
    → ③ 第三人称
      → ④ 疑问句实战
        → ⑤ 现在进行时
          → ⑥ 过去时 & 完成时
            → ⑦ 将来时与情态动词
              → ⑧ 连词与连接
                → ⑨ 描述与比较
                  → ⑩ 从句 & 非谓语动词
```

后一个包应使用前一个包已学的语法，不引入超纲内容。

### 6.4 包内话题一致性

- 同一场景下的话题在句块和词汇上应**有交叉重叠**，形成自然复现
- 不同话题**不得**有完全相同的 warmup_pipeline 内容（禁止复制粘贴）
- 相邻话题使用相近但不同的 targetWord 和句子

### 6.5 难度标注

| 标注 | 含义 | 适用 |
|:-----|:-----|:-----|
| `L1` | 零基础~初级，be 动词、一般现在时、简单句 | ① ② ③ ④ ⑥ ⑧ |
| `L1~L2` | 初级~中级，含进行时、情态动词、比较级等 | ⑤ ⑦ ⑨ ⑩ |

## 七、质量检查清单

创建或修改学习包后，逐项检查以下内容：

### 7.1 CSV 文件检查

- [ ] 所有列名是否正确（snake_case）
- [ ] `scene_title` 在 `scenes.csv` 中都有定义
- [ ] `topic_title` 在 `training_topics.csv` 中都有定义
- [ ] CSV 字段含逗号时已用双引号包裹
- [ ] 无重复的 `title`（包内唯一）
- [ ] 无空行或格式错误

### 7.2 Warmup Pipeline 检查

- [ ] 每个话题至少 3 个练习步骤
- [ ] 没有 `items.length === 1` 的单条目 section
- [ ] `cs_zh_0` 至少 3 条，`cs_en_1` 至少 2 条
- [ ] `pd` 至少 2 条
- [ ] 同一包内，不同话题的 pipeline 内容不重复（无复制粘贴）
- [ ] `targetWord` 存在于 `scene_vocabulary.csv` 中
- [ ] JSON 格式正确，无语法错误
- [ ] 所有 `sd_3` 的 levels 是累进的（每级在前一级基础上增加）
- [ ] 所有 `vsb_1` 的 patterns 在 pipeline 数组内部（不在数组外）
- [ ] 每条 `answer` 是完整的正确翻译

### 7.3 句块检查

- [ ] 所有句块是**完整句子**，不是短语
- [ ] 句块与话题内容匹配

### 7.4 内容一致性检查

- [ ] `training_topics.csv` 的 `title` 与 `warmup_pipeline.json` 的顶层 key 完全一致
- [ ] `scenes.csv` 的 `title` 与 `training_topics.csv` 的 `scene_title` 一致
- [ ] `ink_script_key`（如有）对应到 ink-scripts/ 目录下实际存在的文件
- [ ] 学习包的功能介绍.md 中的数据规模与实际 CSV 数据匹配

### 7.5 自动化验证

```bash
node -e "
const fs = require('fs');
const p = 'apps/backend/prisma/data/packages/<包名>/warmup_pipeline.json';
const w = JSON.parse(fs.readFileSync(p, 'utf8'));
let totalSingle = 0, totalItems = 0;
Object.entries(w).forEach(([topic, t]) => {
  (t.outputTraining.pipeline || []).forEach(s => {
    const items = s.items || [];
    if (items.length === 1) { totalSingle++; console.log('单条目:', topic, s.id); }
    totalItems += items.length;
  });
});
console.log('总条目:', totalItems, '| 单条目 section 数:', totalSingle);
if (totalSingle === 0) console.log('✅ 无单条目问题');
"
```
