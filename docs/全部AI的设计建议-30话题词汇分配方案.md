# 全部 AI 的设计建议：30 话题词汇-句块-句型分配方案

> 基于牛津 3000 词表，面向基础①（Beginner）级别 231 个单词、93 个句块、55 个句型。

---

## 一、设计哲学

### 1.1 两层分配策略

| 层级 | 定义 | 占比建议 | 示例 |
|------|------|----------|------|
| **核心词 (Core)** | 词义与话题直接相关 | 60-70% | `年龄与数字` → old, young, age, number |
| **扩展词 (Extension)** | 词义不直接相关，但能高效练习本话题的句块/句型 | 30-40% | `年龄与数字` → apple（练 "How old" 句型 + 数数场景） |

### 1.2 判断标准：这个词配不配待在这个话题里？

```
问自己三个问题：
1. 这个词跟话题标题有语义关联吗？ → 有 → 核心词
2. 这个词能放进本话题的句块/句型里练吗？ → 能 → 扩展词
3. 如果两个都不满足 → 不该放在这
```

### 1.3 累加式设计 (Cumulative Design)

```
Topic 1-5  自我介绍    → 打好身份词汇基础
Topic 6-10 描述事物    → 复用身份词 + 新增描述词
Topic 11-15 描述他人   → 复用身份词+描述词 + 新增人物词
Topic 16-20 日常寒暄    → 复用前三段 + 新增社交词
Topic 21-25 数字与时间  → 复用前四段 + 新增数量/时间词
Topic 26-30 基本需求    → 复用全部 + 新增需求/情感词
```

**关键原则**：后面的话题中，"扩展词"可以来自前面话题的核心词。用户在 Topic 25 看到 Topic 1 的词不会觉得突兀——反而觉得"这个词我学过！"。

## 三、逐话题分配方案

> 格式：`word` = 核心词，`*word*` = 扩展词（括号内注明服务于哪个句块/句型）

### 场景一：自我介绍（Topics 01-05）

#### Topic 01 · 我是谁（8词）

```
核心: I, name, my, friend, brother, father, mother, she
扩展: people（He is ___ 句型练习）, nurse（I am a ___ 职业替换）
```

| 句块 | 配套 |
|------|------|
| `I am ___` | name, I, brother, father, nurse, friend, people |
| `I am a ___` | student（复用Topic 02）, teacher（复用） |
| `My name is ___` | my, name |
| `He is / She is` | he, she, brother, father, mother |

#### Topic 02 · 你是谁（8词）

```
核心: you, your, different, why, it, then, or, tired
扩展: *he*（Are you? vs Is he? 对比练）, *good*（You are very ___ 句型）
```

#### Topic 03 · 我的信息（9词）

```
核心: afternoon, car, colour, here, hot, night, snow, table, weather
扩展: *name*（复用01，My name is ___. I am from ___ 组合练）
```

#### Topic 04 · 问候与礼貌（8词）

```
核心: good, have, listen, please, read, stand, walk, write
扩展: *friend*（Nice to meet you. I am ___. → 填入friend）
```

**注意**：CSV 中 `$3` 两处数据损坏需修复。

#### Topic 05 · 年龄与数字（7词）

```
核心: old, young, age, year, new, number, how
扩展: *apple, banana*（How many apples? → 数字+名词练数量）
```

> 当前问题：apple, banana, blue, ear, milk 等食物/颜色/身体词塞在这里毫无道理。
> 修复：搬走这些，换入真正的年龄/数字核心词。

### 4.2 每个话题的理想大小

| 话题数 | 建议词数 | 核心:扩展 |
|--------|---------|-----------|
| 30 | 7-9 词 | 60:40 |

当前 231 ÷ 30 ≈ **7.7 词/话题**，基本合理。

### 4.3 句块 & 句型如何驱动词汇分配

句型/句块中出现的**槽位词**（slot words）应该优先成为该话题的扩展词：

| 句型 | 槽位 | 建议扩展词来源 |
|------|------|---------------|
| `I am ___` | 职业/名字 | Topic 01, 13 |
| `It is ___` | 颜色/大小/物品 | Topic 06, 07 |
| `I want ___` | 食物/饮品 | Topic 26 |
| `I like ___` | 任意名词 | Topic 28（引入更多实物词） |
| `Can you ___?` | 动词 | Topic 27, 30 |
| `There is/are ___` | 数量+名词 | Topic 10（练习数数+名词） |

## 六、设计检查清单

每个话题完成后，用这个清单自查：

```
□ 核心词 ≥ 4 个（与话题直接语义相关）
□ 扩展词 ≤ 4 个（能放进本话题的句块/句型练）
□ 每个扩展词都能说清楚"为什么放这"（标注服务的句型）
□ 没有"不知道为什么在这"的词
□ 话题词数 7-9 个
□ 本话题句型槽位被词汇覆盖
□ 能复用前面话题的核心词（累加原则）
```
