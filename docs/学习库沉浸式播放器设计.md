# 学习库沉浸式播放器设计

> 适用范围：移动端 Capacitor App 优先，Web/PWA 降级可用  
> 目标页面：我的学习库、学习包知识点中的单词、句块、句型详情  
> 最后更新：2026-07-10

---

## 1. 背景与目标

现在的「沉浸式」更接近 `LearningInsightDialog`：用户点进某个单词、句块或句型后，查看讲解、结构、例句，并通过底部按钮切换上一条/下一条。新功能要把它升级成一个「场景化学习播放器」：

- 学习库中的单词、句块、句型都可以进入同一套沉浸式播放器。
- 播放器界面沿用 `LearningInsightDialog` / 今日任务 Dialog 的全屏弹窗气质：顶部 header、中间内容区、底部固定功能区。
- 内容不再用 tab 切换，默认只保留「意思/见解 + 一个代表例句」这一套主视图。
- 底部增加播放/暂停、显示/隐藏文案等播放器控制。
- Header 右侧增加设置按钮，位置参考今日任务里的「自动 switch」区域，点击后打开 Drawer。
- Drawer 中配置定时关闭、播放次数、倍速、是否播放译文、是否播放例句译文等。
- Capacitor 移动端锁屏后仍可继续播放，并在锁屏/通知区域展示播放器控制。

一句话：用户可以把学习库当成听力素材一样循环听，边走路、锁屏、通勤时继续复习。

## 2. 用户入口

### 2.1 入口位置

入口优先放在已有「查看/沉浸式」按钮的位置，不额外制造新导航层级。

| 场景 | 入口 |
| --- | --- |
| 学习包单元页 `LearningUnitPage` | 单词、句块、句型卡片展开后，原「查看」按钮文案改为「沉浸式」或图标按钮 |
| 我的学习库 | 单词、句块、句型列表项进入详情时使用同一播放器 |
| 今日任务复习记录 | 后续可复用播放器，但不是第一阶段必须项 |

### 2.2 进入规则

点击某条内容后，播放器队列默认取当前列表上下文：

- 从单词列表进入：队列只包含单词。
- 从句块列表进入：队列只包含句块。
- 从句型列表进入：队列只包含句型。
- 后续可以支持「混合播放」：单词 + 句块 + 句型一起播，但第一版不强行做。

播放器打开时从当前点击项开始播放/待播放，底部仍支持上一条/下一条。

## 3. 信息架构

### 3.1 不再使用 tab

现有 `LearningInsightDialog` 中：

- 单词：词典 / 讲解 / 例句 tab
- 句块：讲解 / 例句 tab
- 句型：结构 / 讲解 / 例句 tab

播放器模式中不再显示 tab，主内容固定为：

1. 核心文本：单词 / 句块 / 句型。
2. 意思或见解：优先展示中文含义，其次展示 AI 讲解摘要。
3. 一个代表例句：优先取第一条例句；句型优先取 `example`；如果没有例句，展示空状态。
4. 可选译文：由底部「显示/隐藏」和播放设置共同控制。

### 3.2 展示优先级

| 类型 | 主标题 | 含义/见解 | 代表例句 |
| --- | --- | --- | --- |
| 单词 `word` | `word` | `meaning`，其次 `definitionEn` 摘要 | `examples[0].en` |
| 句块 `chunk` | `text` | `meaning` + `description` 摘要 | `examples[0].en` |
| 句型 `pattern` | `pattern` | `meaning` + `description` 摘要 | `example`，其次 `examples[0].en` |

主视图不要做成资料页，而要像播放器：当前正在听的内容是视觉中心，讲解只是辅助。

## 4. 播放器界面

### 4.1 整体布局

沿用现有全屏 Dialog：

```tsx
<DialogContent className="h-[100dvh] w-screen ...">
  <ImmersivePlayerHeader />
  <ImmersivePlayerBody />
  <ImmersivePlayerFooter />
</DialogContent>
```

移动端为全屏；桌面端可以继续保持 `md:h-[88vh] md:max-w-3xl md:rounded-2xl`。

### 4.2 Header

Header 保持现有 `InsightHeader` 的信息密度，但右侧增加设置按钮。

建议结构：

- 左侧：类型 icon + 类型 badge。
- 中间：当前内容标题，可两行截断。
- 右侧：
  - 设置按钮：`SlidersHorizontal` 或 `Settings2` 图标。
  - 关闭按钮：`ChevronDown`。

设置按钮位置参考 `TodayTaskPage` 中 header 右侧的自动切换区域：和关闭按钮并排，保持小圆形按钮或轻量 capsule。

### 4.3 Body

Body 是播放器的「封面区」：

- 顶部显示当前序号：`3 / 20`。
- 中央显示核心文本，字号大于普通详情页。
- 下方显示中文意思/见解。
- 再下方显示一个例句块。
- 当「隐藏文案」开启时：
  - 核心文本可保留或轻微弱化。
  - 中文意思、例句译文隐藏。
  - 英文例句是否隐藏由产品策略决定，建议保留英文例句，隐藏中文解释。

### 4.4 Footer

底部仍是固定功能区，但从「查看详情翻页」升级为「播放器控制」。

建议按钮从左到右：

1. 上一条：`ChevronLeft`
2. 播放/暂停：`Play` / `Pause`，作为主按钮
3. 下一条：`ChevronRight`
4. 显示/隐藏文案：`Eye` / `EyeOff`
5. 队列：`ListMusic`

移动端空间有限时，上一条/下一条可以做 icon button，中间播放按钮更突出。底部保留当前进度文本，例如 `3 / 20`。

## 5. 设置 Drawer

### 5.1 入口

Header 右侧设置按钮打开 Drawer：

```tsx
<Drawer open={settingsOpen} onOpenChange={setSettingsOpen}>
  <DrawerContent className="h-auto max-h-[88dvh] rounded-t-2xl">
    <ImmersivePlayerSettings />
  </DrawerContent>
</Drawer>
```

### 5.2 设置项

| 设置 | 控件 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 定时关闭 | segmented control / select | 关闭 | 关闭、5 分钟、10 分钟、15 分钟、30 分钟、播完当前队列 |
| 单条播放次数 | stepper / select | 1 次 | 每个主文本重复几遍后进入下一条 |
| 播放倍速 | slider 或 segmented control | 1.0x | 建议 `0.75x / 1.0x / 1.25x / 1.5x` |
| 播放主文本 | switch | 开启 | 单词/句块/句型本身 |
| 播放译文 | switch | 开启 | 播放 `meaning` |
| 播放例句 | switch | 开启 | 播放代表例句英文 |
| 播放例句译文 | switch | 关闭 | 播放例句中文译文 |
| 自动下一条 | switch | 开启 | 当前播放单元结束后进入下一条 |
| 循环队列 | switch | 关闭 | 队列结束后回到第一条 |

### 5.3 设置持久化

播放器设置应写入 `preferences.store` 或单独的 Zustand persist store。

建议 key：

```ts
type ImmersivePlaybackSettings = {
  sleepTimerMinutes: 0 | 5 | 10 | 15 | 30
  sleepTimerMode?: 'duration' | 'end-of-queue'
  repeatPerItem: 1 | 2 | 3 | 5
  playbackRate: 0.75 | 1 | 1.25 | 1.5
  playMainText: boolean
  playMeaning: boolean
  playExample: boolean
  playExampleTranslation: boolean
  autoNext: boolean
  loopQueue: boolean
  textVisible: boolean
}
```

## 6. 播放队列模型

### 6.1 标准播放项

把单词、句块、句型统一转换成播放器内部模型。

```ts
type ImmersivePlayerItemKind = 'word' | 'chunk' | 'pattern'

type ImmersivePlayerItem = {
  id: string
  kind: ImmersivePlayerItemKind
  title: string
  meaning?: string | null
  insight?: string | null
  exampleEn?: string | null
  exampleZh?: string | null
  audio?: {
    mainTextUrl?: string | null
    exampleUrl?: string | null
  }
  sourceSnapshot?: unknown
}
```

### 6.2 播放片段

每个 item 根据设置展开成多个片段：

```ts
type PlaybackSegment = {
  itemId: string
  role: 'main' | 'meaning' | 'example' | 'exampleTranslation'
  text: string
  audioUrl?: string | null
  lang: 'en' | 'zh'
}
```

例如播放一个句块：

1. `main`: `take it for granted`
2. `meaning`: `认为理所当然`
3. `example`: `Don't take your parents' help for granted.`
4. `exampleTranslation`: `不要把父母的帮助视为理所当然。`

如果用户设置 `repeatPerItem = 3`，只重复主文本，还是重复整个片段组，需要明确。建议第一版：重复整个 item 的片段组，更符合「听一遍完整信息」。

## 7. 音频来源策略

### 7.1 第一阶段：复用现有 Web 音频

当前项目已有：

- `useCachedAudio()`：根据 `audioUrl` / `audioAssetId` 播放并缓存。
- `tts-api.ts`：可合成文本音频。
- 学习包资源：部分 warmup/例句音频已通过 `audioAssetId` 离线缓存。

第一阶段可先在前台播放：

- 如果 `audioUrl` 存在，直接播放。
- 如果只有文本且需要播放，调用后端 TTS 合成并缓存。
- 如果 TTS 失败，则跳过该片段并 toast/静默记录。

### 7.2 第二阶段：统一 TTS 预生成

为了锁屏和后台稳定，建议播放队列在开始前做一次轻量预处理：

- 当前 item 和下一条 item 优先合成。
- 队列其余内容按需后台合成。
- 音频资产保存到现有 `file_assets` / 本地 asset cache 体系。
- 每个片段都尽量有真实音频文件，而不是依赖 WebView 的 `speechSynthesis`。

不要依赖浏览器 `speechSynthesis` 做移动端后台播放；锁屏后可控性和生命周期都不稳定。

## 8. 后台播放与锁屏控制

### 8.1 Web Media Session 的边界

Web 侧可以接入 `navigator.mediaSession`：

- 设置锁屏标题、artist、artwork。
- 注册 play / pause / previoustrack / nexttrack action handler。
- 在支持的浏览器或 WebView 中显示系统媒体控制。

但它不是完整的后台播放方案。MDN 也明确 Media Session API 是让 Web App 向系统媒体 UI 暴露元信息和控制入口，且兼容性并非全平台一致。Capacitor App 锁屏后 WebView 可能被暂停，所以只靠 Web API 不够稳。

### 8.2 Capacitor 原生方案

目标能力：

- App 退到后台或锁屏后，音频继续播放。
- 锁屏/通知区域显示当前学习项。
- 用户可以在锁屏执行播放、暂停、上一条、下一条。
- 原生事件回传给 Web 状态，让 Dialog 前台恢复时状态一致。

需要新增一个原生音频播放桥：

```ts
type ImmersiveAudioPlugin = {
  loadQueue(options: { items: NativePlaybackItem[]; startIndex: number; settings: NativePlaybackSettings }): Promise<void>
  play(): Promise<void>
  pause(): Promise<void>
  stop(): Promise<void>
  next(): Promise<void>
  previous(): Promise<void>
  updateSettings(settings: NativePlaybackSettings): Promise<void>
  getState(): Promise<NativePlaybackState>
}
```

Capacitor 官方文档说明，Plugin 用于把 JavaScript 与原生 API 连接起来；这个功能正好属于 WebView 难以可靠承担、需要原生能力的范围。

### 8.3 iOS 实现要点

iOS 侧需要：

- 开启 Background Modes 中的 Audio。
- 使用 `AVAudioSession` 的 playback 类别。
- 使用 `AVPlayer` / `AVQueuePlayer` 播放音频文件。
- 使用 `MPNowPlayingInfoCenter` 更新锁屏标题、描述、进度。
- 使用 `MPRemoteCommandCenter` 接收锁屏播放、暂停、上一条、下一条。

Apple 官方文档要求媒体后台播放需要配置 Audio 后台模式；否则锁屏/退后台后播放不会稳定持续。

### 8.4 Android 实现要点

Android 侧需要：

- 使用前台服务承载播放，避免后台被系统停止。
- 使用 Media3 / ExoPlayer 播放队列。
- 使用 `MediaSession` 暴露媒体状态与控制。
- 使用媒体通知展示播放状态和控制按钮。

Android 官方文档说明系统媒体控制位于 Quick Settings 附近，并由媒体会话提供可恢复/可控制的播放体验。

### 8.5 降级策略

| 环境 | 能力 |
| --- | --- |
| iOS/Android 原生插件可用 | 后台播放 + 锁屏控制 + 通知控制 |
| Capacitor 但插件未启用 | 前台播放 + 尝试 Media Session |
| Web/PWA | 前台播放 + Media Session 可用时显示系统控制 |

## 9. 状态同步

播放器状态建议集中在一个 store：

```ts
type ImmersivePlayerState = {
  open: boolean
  queue: ImmersivePlayerItem[]
  currentIndex: number
  status: 'idle' | 'loading' | 'playing' | 'paused' | 'ended' | 'error'
  currentSegmentRole?: PlaybackSegment['role']
  textVisible: boolean
  settings: ImmersivePlaybackSettings
}
```

事件流：

1. 用户点击沉浸式。
2. Web 创建 queue，打开 Dialog。
3. Web 根据平台选择 Web 播放器或 Native plugin。
4. 播放状态变化同步到 store。
5. 锁屏按钮触发 native event，Web 收到后更新当前 index/status。
6. App 回到前台时调用 `getState()` 对齐状态。

## 10. 与现有代码的落点

建议新增文件：

```txt
apps/frontend/src/features/learning/components/immersive-player/
  immersive-player-dialog.tsx
  immersive-player-header.tsx
  immersive-player-body.tsx
  immersive-player-footer.tsx
  immersive-player-settings-drawer.tsx
  immersive-player-queue-drawer.tsx
  immersive-player.types.ts
  immersive-player.mapper.ts
  immersive-player.store.ts
  immersive-playback.service.ts
```

建议保留 `LearningInsightDialog`，不要直接改成播放器。原因：

- 详情查看和自动播放是两种不同心智。
- 现有今日任务、学习包详情仍可能需要完整 tab 详情。
- 播放器可以复用 `LearningInsightItem` 类型和 `InsightHeader` 的视觉语言，但不背负旧组件复杂度。

第一阶段接入点：

- `LearningUnitPage`：把 `openDialog()` 旁边新增 `openImmersivePlayer()`，先从同一组 dialog items 映射 queue。
- 我的学习库页面：使用同一 mapper。
- `useCachedAudio()`：可以继续作为 Web 前台播放 fallback，但播放器最好封装一层 `immersive-playback.service.ts`。

## 11. 分阶段实施

### Phase 1：前台播放器 MVP

- 新增沉浸式播放器 Dialog。
- 实现单词、句块、句型统一队列。
- 去掉 tab，只显示意思/见解 + 一个例句。
- 实现底部播放/暂停、上一条、下一条、显示/隐藏、队列按钮。
- 实现设置 Drawer 和设置持久化。
- 前台播放复用现有音频/TTS 能力。

验收：

- 三类内容都能打开播放器。
- 播放顺序和 UI index 正确。
- 隐藏文案后中文解释和译文不显示。
- 设置关闭再打开仍保留。

### Phase 2：音频资产预处理

- 播放前把缺失音频的片段按需 TTS。
- 当前项和下一项优先合成。
- 支持离线学习包中已缓存音频优先。
- 播放失败可跳过片段，不阻断整个队列。

验收：

- 弱网下已有本地音频仍能播放。
- TTS 缓存命中后不重复请求。
- 长队列不会一次性阻塞 UI。

### Phase 3：原生后台播放

- 新增 Capacitor 本地插件或 app 内 native bridge。
- iOS 接入后台音频、锁屏 Now Playing、远程控制。
- Android 接入 Foreground Service、MediaSession、媒体通知。
- Web store 与 native state 双向同步。

验收：

- 锁屏后音频继续。
- 锁屏能看到当前学习项。
- 锁屏可播放/暂停/上一条/下一条。
- 回到 App 后当前项和播放状态准确。

### Phase 4：体验增强

- 支持混合队列。
- 支持「只听收藏」「只听未掌握」「随机播放」。
- 支持后台播放结束后的学习记录。
- 支持迷你播放器，离开 Dialog 后继续前台播放。

## 12. 风险与决策

| 风险 | 决策 |
| --- | --- |
| WebView 锁屏后被暂停 | 后台播放必须走原生，不把 `speechSynthesis` 当主方案 |
| TTS 实时合成慢 | 当前项 + 下一项优先预生成，其余按需 |
| Dialog 旧逻辑过重 | 新建播放器组件，复用数据类型/视觉语言 |
| 设置项过多 | 第一版只展示核心项，高级项后续折叠 |
| iOS 审核 | 后台音频必须是真的用户可感知学习音频，不播放静音保活 |
