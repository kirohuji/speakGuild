# 学习本分组设计方案

> 目标页面：我的学习库  
> 核心改动：将当前平铺的学习库升级为“学习本列表 → 学习本内容”的两级结构。  
> 内容范围：每个学习本可同时收纳单词、句块和句型。

## 1. 背景

当前学习库中的单词、句块和句型分别展示在列表中，所有同类型内容都堆叠在一起。随着收藏内容增加，用户难以按照场景、课程或个人目标维护学习内容。

本方案引入“学习本”：

- 用户可以创建多个学习本，例如“旅行英语”“工作交流”“容易混淆”。
- 一个学习本可以同时包含单词、句块和句型。
- 一个内容可以加入多个学习本。
- 同一内容加入不同学习本后，各学习本分别维护学习状态和复习进度，互不影响。
- 当前学习库内容、详情弹窗、沉浸式播放和复习状态逻辑继续复用。

“学习本”是内容的组织方式，“学习中 / 复习中 / 已掌握”是内容的学习进度，两者相互独立。

## 2. 产品目标

### 2.1 用户目标

- 按场景、课程或个人用途整理学习内容。
- 快速找到某一组相关的单词、句块和句型。
- 收藏内容时直接选择目标学习本。
- 安全地修改或删除学习本，不误删学习记录。

### 2.2 设计目标

- 保留现有学习库二级页面的主要交互和视觉结构。
- 一级页面只负责浏览和管理学习本，避免叠加过多筛选栏。
- 单词、句块和句型使用统一的学习本关系模型。
- 支持 Web、原生端本地存储及离线同步。
- 对现有用户无损迁移，历史内容不能丢失。

### 2.3 非目标

第一期不包含：

- 学习本嵌套或文件夹层级。
- 学习本公开分享、协作编辑。
- 为同一内容在不同学习本中保存不同的掌握状态。
- 自动通过 AI 创建或整理学习本。
- 学习本独立的复习算法。

## 3. 核心概念

### 3.1 学习本

学习本是用户创建的内容集合，可以同时包含：

- 单词 `word`
- 句块 `chunk`
- 句型 `scene_phrase`

例如：

```text
旅行英语
├─ 单词 42
├─ 句块 18
└─ 句型 7
```

### 3.2 系统视图

系统提供两个固定入口：

#### 全部内容

展示用户所有学习本中的学习条目。

- 不可改名。
- 不可删除。
- 不参与学习本排序。
- 它是聚合查询视图，不创建真实学习本记录。
- 同一内容存在于多个学习本时，在“全部内容”中按学习条目分别计算；界面应显示其所属学习本，不能将不同进度错误合并。

#### 未分类

承接用户尚未归入自定义学习本的内容。

- 不可改名。
- 不可删除。
- 为了让未分类内容也拥有独立进度，“未分类”在数据层是一个系统学习本，而不只是 `NOT EXISTS` 查询。
- 用户选择“保存到未分类”时，创建一条属于未分类学习本的学习条目。
- 将未分类内容移动到自定义学习本时，可以转移该学习条目及进度。
- 将未分类内容添加到自定义学习本时，保留未分类条目，并在目标学习本中新建一份独立进度。

### 3.3 学习状态

每条“学习本内容”关系拥有自己独立的学习状态：

- 学习中 `learning`
- 复习中 `reviewing`
- 已掌握 `mastered`

例如，同一个句块同时加入“旅行英语”和“口语高频”：

- 在“旅行英语”中可以是“已掌握”。
- 在“口语高频”中可以仍是“学习中”。
- 两边分别记录复习次数、难度系数、最后复习时间和下次复习时间。
- 在一本学习本中复习或修改状态，不影响另一本学习本。

内容本身只保存词义、例句、句型结构等知识数据；学习进度属于该内容在某一本学习本中的学习实例。

## 4. 信息架构

### 4.1 页面层级

```text
我的学习库
├─ 全部内容
├─ 未分类
├─ 自定义学习本 A
├─ 自定义学习本 B
└─ 自定义学习本 C
    ├─ 单词
    │  ├─ 学习中
    │  ├─ 复习中
    │  └─ 已掌握
    ├─ 句块
    │  ├─ 学习中
    │  ├─ 复习中
    │  └─ 已掌握
    └─ 句型
       ├─ 学习中
       ├─ 复习中
       └─ 已掌握
```

### 4.2 路由建议

```text
/library                         学习本列表
/library/all                     全部内容
/library/uncategorized           未分类内容
/library/notebooks/:notebookId   自定义学习本内容
```

二级页面可通过查询参数保存当前内容类型和学习状态：

```text
/library/notebooks/:notebookId?type=word&status=learning
```

深链兼容建议：

- 保留现有通过 `word`、`chunk`、`pattern` 打开详情的能力。
- 没有学习本上下文的深链默认进入“全部内容”。
- 从某个学习本发起的详情跳转应保留 `notebookId`，关闭详情后返回原列表。

## 5. 一级页面：学习本列表

### 5.1 页面结构

学习本使用纵向单列列表，每个学习本独占一整行，不使用双列宫格。这样名称、内容统计和操作入口都有稳定空间，也更符合“从书架中选择一本学习本”的感觉。

```text
我的学习库                              ＋

┌────────────────────────────────────┐
│ ┌──────┐  全部内容               128 › │
│ │ 全部  │  83 单词 · 31 句块 · 14 句型 │
│ │ 内容  │                              │
│ └──────┘                              │
└────────────────────────────────────┘

┌────────────────────────────────────┐
│ ┌──────┐  未分类                  36 › │
│ │ 待整  │  还没有放入学习本的内容       │
│ │ 理    │                              │
│ └──────┘                              │
└────────────────────────────────────┘

我的学习本

┌────────────────────────────────────┐
│ ┌──────┐  旅行英语                    │
│ │ TRIP │  67 项内容                    │
│ │      │  42 单词 · 18 句块 · 7 句型  ···│
│ └──────┘                              │
└────────────────────────────────────┘

┌────────────────────────────────────┐
│ ┌──────┐  工作交流                    │
│ │ WORK │  42 项内容                    │
│ │      │  25 单词 · 12 句块 · 5 句型  ···│
│ └──────┘                              │
└────────────────────────────────────┘
```

#### 学习本列表项视觉规范

每一行由左侧“书本封面”、中部信息区和右侧操作区组成：

```text
┌─────────────────────────────────────────┐
│  ╭──────╮   学习本名称                    ··· │
│  │      │   67 项内容                       │
│  │ 封面 │   42 单词 · 18 句块 · 7 句型       │
│  ╰──────╯                                  │
└─────────────────────────────────────────┘
```

- 整行宽度占满内容区域，推荐最小高度为 `88px`。
- 左侧书本封面推荐约 `52 × 64px`，需要明显呈现封面、书脊和页边，而不是普通方形图标。
- 书脊可使用一条更深的竖向色带，页边使用浅色细线或轻微错层阴影，形成“实体学习本”的感觉。
- 封面保持简洁，可显示学习本名称的首字、简短英文缩写或系统图形，不要求用户上传图片。
- 每个自定义学习本可从一组克制的封面颜色中稳定取色；颜色一旦生成便跟随学习本保存，不能每次渲染随机变化。
- 中部第一行是学习本名称，使用页面内最强文字层级。
- 第二行显示内容总数。
- 第三行显示单词、句块和句型数量；数量为零的类型仍可保留或按空间省略。
- 右侧 `···` 只打开管理 Drawer，不触发进入学习本。
- 除 `···` 外的整行区域均可点击进入学习本，点击区域至少满足 `44 × 44px`。
- 列表项之间保持适度间距，不使用厚重边框；主要通过背景层次、书本阴影和留白区分。
- 按下时整行轻微缩小或降低亮度，书本封面可产生非常轻微的向右位移，强化“抽出一本书”的反馈。

“全部内容”和“未分类”同样使用书本造型，以保证列表统一，但采用固定封面样式：

- 全部内容：较中性的主色封面，可使用叠放书页或合集标识。
- 未分类：低饱和中性色封面，可使用待整理书签标识。
- 系统视图不显示 `···`，右侧显示进入箭头。

### 5.2 页面操作

- 点击“全部内容”：进入全部内容二级页面。
- 点击“未分类”：进入未分类二级页面。
- 点击自定义学习本：进入对应二级页面。
- 点击右上角 `＋`：打开“新建学习本”Drawer。
- 点击学习本的 `···`：打开“管理学习本”Drawer。
- 返回一级页面时恢复之前的滚动位置。

### 5.3 排序

第一期推荐采用用户手动排序字段 `sortOrder`，默认规则为：

1. 全部内容。
2. 未分类，仅在数量大于 0 时显示。
3. 自定义学习本按照 `sortOrder` 升序。
4. `sortOrder` 相同时按创建时间升序。

第一期可以暂不提供拖拽排序界面，但数据模型预留 `sortOrder`。

### 5.4 空状态

用户没有创建学习本时：

```text
还没有学习本

把单词、句块和句型按场景整理，
复习时会更容易找到。

[创建第一个学习本]
```

“全部内容”入口始终保留。

## 6. 二级页面：学习本内容

### 6.1 页面结构

二级页面复用当前学习库页面的内容列表、详情弹窗和沉浸式播放器：

```text
‹ 学习库              旅行英语          ···

单词 42          句块 18          句型 7

学习中          复习中          已掌握
────────────────────────────────────
当前筛选条件下的内容列表
```

### 6.2 顶部标题

- 自定义学习本：显示学习本名称和管理入口。
- 全部内容：标题显示“全部内容”，不显示管理入口。
- 未分类：标题显示“未分类”，不显示管理入口。

学习本名称过长时单行截断，完整名称可在管理 Drawer 中查看和编辑。

### 6.3 内容类型 Tab

保留现有三个 Tab：

- 单词
- 句块
- 句型

Tab 标签可展示该学习本中对应类型的数量。切换内容类型时：

- 保留当前学习本。
- 默认继续使用当前学习状态。
- 如果产品希望保持现有行为，也可以切换类型时回到“学习中”；应在实现前统一规则。

建议保留状态，减少反复操作。

### 6.4 学习状态筛选

保留现有：

- 学习中
- 复习中
- 已掌握

状态切换只改变列表查询，不改变内容与学习本的关系。

### 6.5 沉浸式播放

从学习本二级页面打开沉浸式播放器时，播放队列只包含当前条件下的内容：

```text
当前学习本 + 当前内容类型 + 当前学习状态
```

例如，在“旅行英语 / 句块 / 复习中”点击沉浸式播放，只播放该列表中的句块。

## 7. Drawer 交互

新建、重命名、删除及内容归类统一使用现有底部 Drawer 组件。

所有 Drawer 必须包含 `DrawerTitle`；如果标题不需要视觉展示，使用 `sr-only` 隐藏。

### 7.1 新建学习本

入口：

- 学习库一级页面右上角 `＋`。
- “保存到学习本”Drawer 中的“新建学习本”。

内容：

```text
新建学习本

名称
[ 例如：旅行英语                     ]

学习本可以同时收纳单词、句块和句型。

[取消]                         [创建]
```

规则：

- 名称去除首尾空格。
- 名称不能为空。
- 建议限制为 30 个字符。
- 同一用户下名称不允许重复，比较时忽略首尾空格和大小写。
- 创建成功后立即进入或选中新学习本，具体取决于入口。
- 从保存流程创建时，创建成功后自动勾选新学习本，不中断原保存流程。

### 7.2 管理学习本

入口：

- 一级页面学习本卡片的 `···`。
- 二级页面右上角 `···`。

内容：

```text
管理“旅行英语”

修改名称
管理内容
删除学习本
```

“删除学习本”使用危险色，但不在首次点击时立即执行。

### 7.3 修改名称

```text
修改学习本名称

[ 旅行英语                          ]

[取消]                         [保存]
```

规则与新建名称一致。保存成功后：

- 一级页面卡片立即更新。
- 二级页面标题立即更新。
- 不改变路由，因为路由使用学习本 ID。

### 7.4 删除学习本

```text
删除“旅行英语”？

该学习本中有 67 项内容。
删除后，该学习本中的学习进度也会被删除。
相同内容在其他学习本中的进度不受影响。

[取消]                   [删除学习本]
```

删除规则：

- 软删除学习本。
- 删除或软删除该学习本的全部关联关系。
- 删除该学习本关系上保存的学习状态、复习次数和复习时间。
- 不删除底层知识内容，也不影响相同内容在其他学习本中的学习实例。
- 删除当前正在浏览的学习本后返回学习库一级页面。
- 删除操作需要支持离线记录和后续同步。

### 7.5 保存到学习本

原有所有添加入口统一使用同一套流程，包括：

- 添加单词。
- 添加句块。
- 添加句型。
- 从内容详情、学习包、练习结果或其他页面点击“加入学习库”。

点击后打开底部 Drawer：

```text
保存到学习本

✓ 旅行英语                    上次使用
□ 工作交流
□ 容易混淆

＋ 新建学习本

[取消]                         [保存]
```

规则：

- 支持多选。
- 已经加入的学习本默认勾选；每个勾选项代表一份独立的学习实例。
- 系统记住用户上一次实际保存内容时使用的学习本。
- 新内容打开 Drawer 时，如果用户没有主动选择其他学习本，默认勾选上一次使用的学习本。
- 用户直接点击“保存”，内容保存到默认勾选的上一次学习本。
- 用户主动选择其他学习本并保存后，最后一次主动选择的学习本成为新的默认学习本。
- 用户一次勾选多个学习本时，最后一个被主动勾选的学习本作为下次默认项；本次内容仍保存到所有已勾选项。
- 默认学习本是用户级偏好，不区分单词、句块和句型，因为同一个学习本可以收纳三类内容。
- Drawer 中为默认项显示轻量的“上次使用”提示，避免用户不知道内容将保存到哪里。
- 用户可以取消全部勾选；此时保存按钮文案改为“保存到未分类”，内容进入学习库但不建立学习本关联。
- 用户没有学习本时，主要操作显示“新建学习本”。
- 用户首次使用且没有上一次选择记录时：
  - 如果已有学习本，默认选中排序第一的自定义学习本。
  - 如果没有学习本，Drawer 引导新建；用户也可以选择“暂不分类”。
- 从当前 Drawer 新建学习本后，新学习本自动勾选，并成为本次保存后的默认学习本。
- 上一次使用的学习本被删除、失效或不再可访问时，清除该偏好，并按照首次使用规则回退。
- 保存内容、创建独立学习实例和更新“上次使用的学习本”应视为一个完整业务操作。
- 离线时先写入本地内容、学习本关系和同步队列。

#### 再次管理已收藏内容

对已经存在于学习库中的内容再次打开 Drawer 时：

- 展示该内容当前所属的全部学习本，并默认勾选。
- 取消某个勾选并保存，表示删除该学习本中的学习实例及其独立进度，需要明确提示。
- 新增勾选并保存，表示在对应学习本中新建学习实例，初始状态为“学习中”。
- 如果取消全部自定义学习本并选择“保存到未分类”，则在系统未分类学习本中创建独立学习实例。

#### 默认学习本偏好

建议在用户偏好或本地设置中保存：

```ts
type LearningNotebookPreference = {
  lastUsedNotebookId: string | null
  updatedAt: string
}
```

该偏好应支持本地持久化，保证 App 重启后仍能沿用。是否跨设备同步可以分阶段实现：

- 第一期：保存在当前设备本地，离线立即可用。
- 后续：同步到用户偏好，使不同设备共享最近使用的学习本。

默认学习本只负责 Drawer 的初始选中状态，最终归属始终以用户点击“保存”时的勾选结果为准。

### 7.6 单个内容管理

建议将当前列表项中较多的右侧操作收拢到 `···` Drawer：

```text
管理内容

添加到其他学习本
从“旅行英语”移除
更改学习状态
从学习库删除
```

区别：

- “从当前学习本移除”删除当前学习实例及其进度，不影响其他学习本。
- “从学习库全部删除”删除该内容在所有学习本中的实例及各自进度，需要更强的二次确认。
- 从当前学习本移除后不会自动复制到未分类；如需保留当前进度，应使用“移动到未分类”。

## 8. 批量管理

第一期建议支持基础批量管理，避免用户只能逐条整理历史内容。

入口：二级页面管理菜单中的“管理内容”。

进入选择模式后支持：

- 全选当前列表。
- 添加到学习本。
- 从当前学习本移除。
- 更改学习状态。
- 从学习库删除。

批量操作只作用于当前列表中被选中的内容，不默认跨内容类型或跨学习状态。

从“全部内容”或“未分类”进入批量管理时，可以使用“添加到学习本”；不存在“从当前学习本移除”。

## 9. 数据模型

现有 `ExpressionItem` 已统一承载单词、句块和句型，因此学习本关系仍可关联 `ExpressionItem`，不要为三种内容分别建表。

但现有 `ExpressionItem` 上的 `masteryStatus`、`reviewCount`、`easeFactor`、`lastReviewedAt` 和 `nextReviewAt` 不能继续作为最终进度来源。新版中这些字段应迁移到 `LearningNotebookItem`，因为进度属于“某个内容在某本学习本中的学习实例”。

### 9.1 Prisma 模型建议

```prisma
model LearningNotebook {
  id        String   @id @default(cuid())
  user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  userId    String
  name      String
  kind      LearningNotebookKind @default(custom)
  sortOrder Int      @default(0)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  deletedAt DateTime?

  items LearningNotebookItem[]

  @@index([userId, sortOrder])
  @@index([userId, updatedAt])
  @@index([userId, deletedAt])
  @@map("learning_notebook")
}

model LearningNotebookItem {
  id               String   @id @default(cuid())
  notebook         LearningNotebook @relation(fields: [notebookId], references: [id], onDelete: Cascade)
  notebookId       String
  expressionItem   ExpressionItem   @relation(fields: [expressionItemId], references: [id], onDelete: Cascade)
  expressionItemId String
  masteryStatus   String   @default("learning")
  reviewCount     Int      @default(0)
  easeFactor      Float    @default(2.5)
  lastReviewedAt  DateTime?
  nextReviewAt    DateTime?
  createdAt        DateTime @default(now())
  updatedAt        DateTime @updatedAt
  deletedAt        DateTime?

  @@unique([notebookId, expressionItemId])
  @@index([expressionItemId])
  @@index([updatedAt])
  @@index([deletedAt])
  @@map("learning_notebook_item")
}

enum LearningNotebookKind {
  custom
  uncategorized
}
```

`ExpressionItem` 增加：

```prisma
notebookItems LearningNotebookItem[]
```

`User` 增加：

```prisma
learningNotebooks LearningNotebook[]
```

每个用户必须且只能有一个 `kind = uncategorized` 的系统学习本。服务端应通过事务和数据库唯一约束保证这一点。

### 9.2 进度归属

读取和更新学习状态时必须使用 `LearningNotebookItem.id`，不能只使用 `ExpressionItem.id`：

```text
ExpressionItem
└─ 知识内容身份：单词、句块或句型

LearningNotebookItem
├─ 所属学习本
├─ 所属知识内容
├─ 学习状态
├─ 复习次数
├─ 难度系数
├─ 最后复习时间
└─ 下次复习时间
```

同一个 `ExpressionItem` 可以对应多条 `LearningNotebookItem`，每条关系的进度完全独立。

### 9.3 名称唯一性

由于需要兼容软删除和大小写不敏感比较，不建议仅依赖简单的 Prisma `@@unique([userId, name])`。

服务层创建和改名时应：

1. 对名称执行 `trim`。
2. 使用大小写不敏感查询检查同一用户的有效学习本。
3. 数据库层根据实际数据库能力增加规范化名称或部分唯一索引。

可选字段：

```prisma
normalizedName String
```

有效学习本使用 `userId + normalizedName` 保证唯一。

响应中的每一项必须同时返回：

示例：


服务端必须校验：

- 学习本属于当前用户。
- 内容属于当前用户。
- 学习本和内容均未被软删除。
- 重复添加保持幂等。

## 11. 离线存储与同步

当前学习库优先读取本地 `expression_entries`，学习本功能必须同时支持 IndexedDB、SQLite 和服务端同步。

### 11.1 本地实体

建议新增：

```text
learning_notebooks
learning_notebook_items
```

本地学习本字段：

```ts
type LocalLearningNotebook = {
  id: string
  remoteId?: string | null
  name: string
  normalizedName: string
  sortOrder: number
  createdAt: string
  updatedAt: string
  deletedAt?: string | null
  syncStatus: 'synced' | 'pending_create' | 'pending_update' | 'pending_delete'
}
```

本地关联字段：

```ts
type LocalLearningNotebookItem = {
  id: string
  notebookId: string
  expressionEntryId: string
  remoteId?: string | null
  masteryStatus: 'learning' | 'reviewing' | 'mastered'
  reviewCount: number
  easeFactor: number
  lastReviewedAt?: string | null
  nextReviewAt?: string | null
  createdAt: string
  updatedAt: string
  deletedAt?: string | null
  syncStatus: 'synced' | 'pending_create' | 'pending_delete'
}
```

### 11.2 本地索引

至少需要：

```text
learning_notebooks(user_id, sort_order)
learning_notebooks(remote_id)
learning_notebooks(sync_status)

learning_notebook_items(notebook_id)
learning_notebook_items(expression_entry_id)
learning_notebook_items(remote_id)
learning_notebook_items(sync_status)
```

### 11.3 离线操作

离线时允许：

- 创建学习本。
- 修改名称。
- 删除学习本。
- 添加或移除内容。

操作立即更新本地 UI，并写入同步队列。恢复网络后按照以下顺序同步：

1. 创建或更新学习本。
2. 创建尚未同步的 `ExpressionItem`。
3. 建立学习本与内容的关联。
4. 同步关联删除。
5. 同步学习本删除。

关联同步必须能够处理本地临时 ID 到远端 ID 的映射。

学习状态变更也必须作为 `learning_notebook_item` 的更新同步，不能再把不同学习本的进度合并回同一条本地 `expression_entries`。

### 11.4 冲突策略

第一期采用“服务端更新时间较新者优先”，但本地未同步操作不能被远端旧数据覆盖。

- 本地 `pending_create/update/delete` 优先保留，直至同步完成。
- 学习本被其他设备删除后，本地关系一并失效。
- 同名创建冲突时保留服务端已有学习本，并将本地关系合并到该学习本。
- 关联新增和删除按 `updatedAt` 或同步事件顺序解决。

## 12. 历史数据迁移

上线后，系统为每个现有用户创建一个不可见管理的“未分类”系统学习本，并将历史内容迁移为该学习本中的学习实例。

推荐迁移体验：

1. 自动创建一个 `kind = uncategorized` 的系统学习本，但不允许用户改名或删除。
2. 为每条历史 `ExpressionItem` 创建对应的 `LearningNotebookItem`。
3. 将历史 `ExpressionItem` 上的学习状态、复习次数、难度系数和复习时间复制到新建的未分类学习实例。
4. 首次进入新版学习库时显示一次引导：

```text
学习库现在支持学习本了

你原来的单词、句块和句型都在“未分类”中，
学习记录和复习进度不会受到影响。

[知道了]   [去整理]
```

5. “去整理”进入未分类页面的批量管理模式。

用户将历史条目“移动”到某个自定义学习本时，保留迁移后的进度；使用“添加到学习本”时，则在目标学习本创建初始状态为“学习中”的新实例。

这种方式不修改历史 `ExpressionItem`，迁移风险最低。

## 13. 删除语义

必须明确区分三种删除：

| 操作 | 删除学习本 | 删除学习实例 | 底层知识内容 | 其他学习本进度 |
|---|---:|---:|---:|---:|
| 从当前学习本移除 | 否 | 当前实例及进度 | 保留 | 不受影响 |
| 移动到其他学习本 | 否 | 转移当前实例 | 保留 | 不受影响 |
| 删除学习本 | 是 | 该学习本全部实例及进度 | 保留 | 不受影响 |
| 从学习库全部删除 | 否 | 该内容全部实例及进度 | 可软删除 | 全部删除 |

所有 UI 文案和接口行为都必须遵守这一区分。

## 14. 权限与安全

- 所有学习本操作必须从登录会话获取 `userId`，不能信任客户端传入的用户 ID。
- 查询、修改和删除学习本时必须校验所有权。
- 添加关联时同时校验学习本和内容的所有权。
- 删除学习本和批量删除内容需要服务端事务。
- 列表数量统计只计算未软删除的内容和有效关联。

## 15. 性能要求

- 学习本一级页面应通过一次接口返回学习本和数量汇总，避免每个卡片单独请求。
- 二级页面继续分页，不一次加载整个学习本。
- 数量汇总按有效关联、内容类型和软删除状态计算。
- 为学习本关系的两端建立索引。
- “未分类”查询使用系统学习本 ID 和学习实例索引，不再使用 `NOT EXISTS` 推导。
- 本地列表按学习本 ID、内容类型和学习状态过滤时应走索引或内存中受控的数据集合。

## 18. 验收标准

### 学习本

- 用户可以创建、改名和删除学习本。
- 同一用户不能创建重名学习本。
- 学习本可同时包含单词、句块和句型。
- 一个内容可以加入多个学习本。
- 一级页面数量统计正确。

### 内容与进度

- 同一内容在不同学习本中可以拥有不同学习状态。
- 在任意学习本中修改状态后，其他学习本的状态和复习数据不发生变化。
- 同一内容在不同学习本中的复习次数、难度系数和下次复习时间独立计算。
- 删除学习本会删除该学习本内的独立进度，但不影响底层知识内容和其他学习本。
- “移动到其他学习本”可以保留当前实例的学习进度。
- “添加到其他学习本”会创建一份初始为“学习中”的新进度。
- 从学习库全部删除内容后，所有学习本中均不再展示。

### 离线与同步

- 离线时可以创建、改名、删除学习本。
- 离线时可以添加和移除内容。
- 恢复网络后数据正确同步。
- 多设备同步后不产生重复学习本关系。
- 历史内容升级后全部出现在“未分类”，且学习记录不丢失。

### 页面体验

- 一级页面点击学习本可进入当前学习库内容页。
- 二级页面保留单词、句块、句型及学习状态筛选。
- 原有添加单词、句块和句型的入口均打开“保存到学习本”Drawer。
- Drawer 默认选中用户上一次使用的学习本。
- 用户不修改选择并直接保存时，内容进入上一次使用的学习本。
- 新建学习本后，新学习本自动选中并成为新的默认学习本。
- 默认学习本被删除后能够正确回退，不会导致保存失败。
- 用户取消全部选择时，内容可以保存到“未分类”。
- Drawer 均具备明确标题、取消和确认操作。
- 返回一级页面后恢复滚动位置。
- 沉浸式播放器只播放当前列表范围内的内容。

## 19. 最终决策摘要

- 产品名称统一使用“学习本”，不再使用“单词本”。
- 学习库改为“学习本列表 → 学习本内容”的两级页面。
- 每个学习本可同时包含单词、句块和句型。
- 一个内容可以属于多个学习本。
- 学习状态和复习进度属于内容在某一本学习本中的学习实例，各学习本相互独立。
- “全部内容”是聚合查询视图；“未分类”是不可改名、不可删除的系统学习本。
- 新建、改名、删除和归类使用底部 Drawer。
- 所有添加单词、句块和句型的入口统一通过 Drawer 选择或新建学习本。
- Drawer 默认使用用户上一次选择的学习本；没有重新选择时直接沿用。
- 删除学习本会删除该本中的独立进度，但不删除底层知识内容，也不影响其他学习本。
- 历史内容无损进入“未分类”。
- 学习本及关系必须支持本地离线存储和双向同步。
