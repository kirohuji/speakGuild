# 漫语町 生产运营手册

> 本手册整合上线检查清单、稳定性方案与 App Store 截图指南，涵盖从审核合规到日常运维的全流程。
> 最后更新：2026-06-14

---

## 一、上线前检查清单

### 1.1 账号注销闭环

Apple 要求：如果 App 支持创建账号，就必须允许用户在 App 内发起账号删除。

当前项目已有后端 `/auth/delete-account` 和前端注销入口，需确认：

- 邮箱/密码注册用户可以正常注销。
- Apple 登录、微信登录等第三方登录用户也可以注销。
- 没有密码的第三方登录用户，不应被"请输入密码"卡住。
- 注销后会删除或匿名化学习记录、收藏、生词本、练习记录。
- 法律或财务要求保留的数据（如订单记录）应有明确保留策略。

### 1.2 iOS 付费只走 Apple IAP / RevenueCat

- iOS 包内只接入 Apple Store 订阅，通过 RevenueCat 完成购买、恢复购买和订阅状态同步。
- iOS App 内不展示支付宝、微信支付购买入口，不引导外部支付。
- 会员页提供 Apple IAP 商品、"恢复购买"入口、"管理订阅"入口。
- 支付宝和微信支付代码可保留给 Web/Android。
- 会员判权统一以后端 `user_membership` 为准，RevenueCat webhook 负责同步。

### 1.3 App Store 隐私合规

需要准备并对齐：

- App Store Connect 隐私标签。
- `PrivacyInfo.xcprivacy` 隐私清单（已添加 `apps/frontend/ios/App/App/PrivacyInfo.xcprivacy`）。
- Required Reason API 声明。
- 第三方 SDK 隐私说明（RevenueCat、Capacitor 插件、Sentry）。

当前清单覆盖：

- Required Reason API：`UserDefaults`、文件时间戳、磁盘空间
- 数据类型：姓名、邮箱、手机号、用户 ID、录音音频、用户内容、购买记录、崩溃数据、性能数据
- 不声明追踪：`NSPrivacyTracking=false`

App Store Connect 隐私问卷建议保持一致：

| 数据 | 是否关联用户 | 用途 |
|---|---:|---|
| 姓名、邮箱、手机号、用户 ID | 是 | App 功能、账号管理、客服 |
| 录音音频、用户输入内容 | 是 | 语音识别、AI 测评、学习反馈 |
| 购买记录 | 是 | 订阅权益、恢复购买、客服核查 |
| 崩溃数据、性能数据 | 否 | 稳定性监控 |

提交前在 Xcode 里生成 Privacy Report，对照第三方 SDK 隐私说明。

## 二、崩溃与错误监控

已实现监控上报：

- 前端 `window.error`、`unhandledrejection`、接口 5xx → 后端 `/client-errors`
- 配置 `VITE_SENTRY_DSN` → `@sentry/capacitor` + `@sentry/react` 上报 Sentry/GlitchTip
- 后端 5xx → 全局异常过滤器触发告警
- 配置 `SENTRY_DSN` → Sentry-compatible envelope 同步上报

环境变量：

```env
VITE_SENTRY_DSN=
SENTRY_DSN=
```

| 变量 | 用途 | 不配置时 |
|---|---|---|
| `VITE_SENTRY_DSN` | 前端/App 错误上报 Sentry/GlitchTip | 仍发到后端 `/client-errors` |
| `SENTRY_DSN` | 后端错误同步上报 Sentry/GlitchTip | 只写后端日志 + webhook |

自托管说明：

- 可自托管 Sentry（Docker Compose），但组件较多，不建议和业务混在同一 compose。
- GlitchTip 更轻量，适合个人或小团队。
- 低配自托管只开错误监控，不开启 Replay、Profiling、高采样性能追踪。

## 三、运营告警

已实现轻量 webhook 告警服务，默认关闭，配置后启用：

```env
OPS_ALERT_WEBHOOK_URL=
OPS_ALERT_WEBHOOK_AUTHORIZATION=
OPS_ALERT_MIN_INTERVAL_SECONDS=300
```

| 变量 | 用途 | 示例 |
|---|---|---|
| `OPS_ALERT_WEBHOOK_URL` | 告警接收地址（飞书/企微/Slack） | `https://open.feishu.cn/open-apis/bot/v2/hook/...` |
| `OPS_ALERT_WEBHOOK_AUTHORIZATION` | webhook 鉴权头 | `Bearer your-secret-token` |
| `OPS_ALERT_MIN_INTERVAL_SECONDS` | 同类告警最小间隔 | `300` |

会触发告警的情况：

- 后端 5xx
- 前端白屏/运行时错误
- RevenueCat webhook 处理失败
- RevenueCat 订阅扣费异常
- 数据库或文件资产容量接近阈值

**健康检查**：

```text
GET /api/v1/manyu/health
```

建议用 UptimeRobot、Better Stack、腾讯云拨测等外部服务监控。

**容量阈值**：

```env
OPS_CAPACITY_CHECK_CRON=0 15 9 * * *
OPS_DB_MAX_BYTES=0
OPS_FILE_ASSET_MAX_BYTES=0
```

| 变量 | 用途 | 示例 |
|---|---|---|
| `OPS_DB_MAX_BYTES` | 数据库套餐上限 | 10GB = `10737418240` |
| `OPS_FILE_ASSET_MAX_BYTES` | 文件资源上限 | 50GB = `53687091200` |

`0` 表示不启用。COS/CDN 真实容量和账单告警仍建议在云厂商控制台配置。

推荐最小配置：

```env
OPS_ALERT_WEBHOOK_URL=你的飞书或企业微信机器人地址
OPS_ALERT_WEBHOOK_AUTHORIZATION=
OPS_ALERT_MIN_INTERVAL_SECONDS=300
OPS_DB_MAX_BYTES=10737418240
OPS_FILE_ASSET_MAX_BYTES=53687091200
```

## 四、内容发布与回滚

保持最小流程：

1. 管理员在后台预览内容、学习包或移动 bundle。
2. 先发 staging 或低灰度 `rolloutPercent=5~20`。
3. 确认无错误后提高灰度到 100。
4. 出错时先把对应 bundle/包 `enabled=false`，不要直接删除。
5. 保留上一版可用包，必要时重新启用旧版本。

规则：

- 只有管理员能发布和停用。
- 发布前必须能预览。
- 线上异常优先停用问题包，再慢慢修内容。
- 删除只用于确认长期不再使用的旧资产。

功能对应：

| 功能 | 用途 |
|---|---|
| `enabled=false` | 快速停用某个移动 bundle 或内容包 |
| `rolloutPercent` | 灰度发布，先给少量用户 |
| `channel=staging/production` | 区分测试通道和正式通道 |
| `minNativeVersion` | 防止旧 App 拉到不兼容的新 bundle |
| 保留旧版本 | 出错时重新启用旧版本实现回滚 |

## 五、客服反馈闭环

已具备：

- 用户提交反馈
- 管理员查看历史
- 管理员回复用户并发送通知
- 反馈状态：待处理、已解决、已关闭
- 联系方式
- 严重问题标记

最小处理规则：

- 崩溃、支付、无法登录、数据丢失：标记严重。
- 严重问题当天处理，普通反馈按批处理。
- 回复用户后，状态改为已解决。

字段说明：

| 字段 | 用途 |
|---|---|
| `status=pending` | 新反馈或还没处理完 |
| `status=resolved` | 已回复或已修复 |
| `status=closed` | 无需继续处理或重复问题 |
| `isCritical=true` | 严重问题，优先处理 |
| `adminNote` | 管理员回复内容，会通知用户 |
| `contact` | 用户留下的联系方式 |

## 六、App Store 截图指南

### 尺寸要求

| 设备 | 竖屏 | 横屏 |
|---|---|---|
| iPhone 6.5" | 1242 × 2688 px | 2688 × 1242 px |
| iPhone 6.7" | 1284 × 2778 px | 2778 × 1284 px |

> 建议只传 **6.7" 尺寸**（1284 × 2778），Apple 会自动缩放适配其他机型。

### 数量与内容

- App 预览视频：最多 3 个（30 秒内）
- 截图：最多 10 张，建议 6-8 张竖屏 + 2 张横屏

| # | 页面 | 路由 | 说明 |
|---|---|---|---|
| 1 | 首页 | `/` | 时钟 + 每日名言 + 签到入口 |
| 2 | 剧本模式 | `/script` | 互动剧本列表 |
| 3 | 探索模式 | `/explore` | 地图 + 地点探索 |
| 4 | 练习记录 | `/learning` | 学习计划 |
| 5 | 个人中心 | `/profile` | 学习统计概览 |
| 6 | 表达库 | `/expressions` | 生词本 / 句块库 |

### 截图方法

**方案 A：Xcode Simulator（推荐）**

```bash
cd apps/backend && pnpm dev
cd apps/frontend && pnpm dev
cd apps/frontend && npx cap run ios
```

模拟器启动后：登录测试账号 → 导航到各页面 → `⌘Cmd + S` 截图。

**方案 B：真机**：Xcode 构建到真机 → 电源键 + 音量上键截图。

### 上传前检查

- [ ] 尺寸正确（1284 × 2778 或 1242 × 2688）
- [ ] 不包含模拟器边框、地址栏
- [ ] PNG 格式，无压缩失真
- [ ] 第一张截图选最能体现核心价值的页面
- [ ] 截图文案不包含占位文本
- [ ] 文字使用简体中文（中国区要求）

## 七、App Review 准备

提交前准备：

- 测试账号和密码。
- 可体验高级功能的审核账号。
- App Review Notes 写清楚主要功能路径。
- 依赖网络/AI/订阅沙盒的功能写清楚测试方式。
- 确保审核环境后端稳定。

## 八、运营初期关注数据

- 新增用户数、日活、周活
- 注册到首次练习转化率、首次练习完成率
- 次日留存、7 日留存
- 每日练习完成数
- 免费用户到会员转化率
- 订阅购买数、取消数、退款数
- AI / TTS / STT 人均成本
- 崩溃率、接口错误率
- 客服反馈数量和处理时长
