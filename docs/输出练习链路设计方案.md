## 1. 核心架构：Output Training Pipeline

### 当前链路

```
学习包 → VN 对话 → AI 判断/复盘
```

### 目标链路

```
Learn     看讲解、词汇、Chunk、句型         ← prepare 阶段（已有）
  ↓
Activate  Chunk 输出热身 + 词汇搭配输出     ← guided 阶段（MVP）
  ↓
Recall    遮挡复述 / 中文意图转英文          ← guided 阶段（增强）
  ↓
Perform   VN 任务型对话（目标闯关）          ← practice 阶段（已有，目标结构化）
  ↓
Repair    AI 纠错后二次输出                 ← practice 阶段内（MVP）
  ↓
Transfer  换场景迁移复练                    ← analysis 阶段后（增强）
  ↓
Review    隔天错句/Chunk/词汇复练           ← ExpressionItem 驱动（MVP）
```

### 两条训练线

| 训练线 | 材料 | 最终目标 |
|--------|------|---------|
| **Chunk 线** | `Chunk` → 替换 → 造句 → VN 使用 | 掌握可复用表达，自动脱口而出 |
| **词汇线** | `Vocabulary` → 搭配 → 句子 → VN 场景 | 从"认识"变成"能在场景里用出来" |

两条线的关系：**词汇进入 Chunk，Chunk 进入句子，句子进入 VN 任务，VN 任务沉淀成复练卡。**

### 7 个阶段 vs 当前代码

| 阶段 | 现有基础 | MVP / 增强 |
|------|---------|-----------|
| **Learn** | `prepare` 页面：教学 Markdown、词汇表、句型、Chunk 列表 | 保持不动 |
| **Activate** | `ChunkActivationPanel` — 展示、展开、收藏 | **MVP**：增加 Chunk 跟读/替换/造句 + 词汇搭配输出 |
| **Recall** | 暂缺 | **增强**：遮挡复述/中文意图转英文 |
| **Perform** | `VnPlayer` + Ink + `judgeDialogueTurn` | **MVP**：`TrainingTopic.metadata` 结构化 objectives |
| **Repair** | AI feedback 已有，只展示不强制重说 | **MVP**：correction + retryRequired + 二次输出 |
| **Transfer** | 暂缺 | **增强**：同 Chunk/词汇换场景练习 |
| **Review** | `ExpressionItem` + SM-2 间隔复习已有 | **MVP**：自动生成复练卡片（错句/Chunk/词汇/升级卡） |

当前 `buildObjectives()` 只做了粗糙的目标生成：根据 `topicPatterns` 生成"使用句型: xxx"，根据标题生成"围绕话题 xxx 展开对话"。

## 4. 改动二：前端新增 guided phase

### 4.1 设计原则：复用 UI，新增业务逻辑

guided 热身**不一定新建独立页面**。优先复用现有单词 / Chunk / 句型 Dialog，只增加练习状态、提交判断、完成进度。

```
现有 Dialog = 内容展示（释义、例句、音频、收藏）
guided      = 内容展示 + 输出任务 + AI 判断 + 进度
```

因此将现有 Dialog 改为多模式：

```typescript
type InsightDialogMode = 'inspect' | 'guided' | 'review'
```

| mode | 用途 | UI 行为 |
|------|------|---------|
| `inspect` | 普通查看 | 展示释义、例句、收藏按钮 |
| `guided` | VN 前热身 | 显示任务、输入/录音、AI 判断、完成按钮 |
| `review` | 复练卡 | 显示原句/改正句、让用户再说一次、更新复习状态 |

### 4.2 Phase 定义

当前：

```typescript
type Phase = 'prepare' | 'practice' | 'analysis'
```

改为：

```typescript
type Phase = 'prepare' | 'guided' | 'practice' | 'analysis'
```

### 4.3 MVP 流程

guided 在移动端不一定是独立 route。推荐方案：

```
PracticeSessionPage
  prepare 阶段
    ↓ 用户点击"开始输出热身"
  打开全屏 Dialog / Bottom Sheet / Drawer（复用现有组件）
    ↓ 完成 2-3 个输出任务
  关闭 Dialog
    ↓ 进入 VN practice
  practice（已有，VN 对话 + targeted_output Judge + retry）
    ↓
  analysis（已有，AI 复盘 + 自动生成 ExpressionItem）
```

移动端 UI 示意：

```
[宿舍入住 · 输出热身]                ← 全屏 Dialog / Drawer
进度 1 / 3

Chunk:
I'd like to...

任务：
请用这个表达说一句：
"我想办理入住。"

[录音 / 输入框]

[提交] [查看提示] [跳过]
```

### 4.4 MVP 组件

#### ChunkOutputDrillCard（可嵌入现有 Dialog）

当前 `ChunkActivationPanel` 以展示为主，每个 Chunk 卡片增加输出操作：

```tsx
<ChunkOutputDrillCard chunk={chunk} mode="guided">
  <Button onClick={→ 跟读}>🔊 跟读一句</Button>
  <Button onClick={→ 替换}>✏️ 换词造句</Button>
  <Button onClick={→ 回答}>💬 用它回答</Button>
</ChunkOutputDrillCard>
```

**交互流程**：

1. **跟读**：播放 Chunk 原文音频 → 用户录音 → 语音识别对比
2. **换词造句**：显示替换候选词 → 用户说出完整句 → 精确/AI 判断（`targeted_output` 模式）
3. **用它回答**：显示微型场景（如 NPC 问 "How can I help you?"）→ 用户必须用目标 Chunk 回答 → AI 判断

#### VocabOutputCard（可嵌入现有 Dialog）

单词不再只展示释义，而是 5 层递进：

```
Word: check in

1. 释义        办理入住 / 登记                           ← 现有数据
2. 常用搭配    check in at the front desk / check into...  ← collocations
3. 替换造句    我想办理入住。→ I'd like to check in.       ← pipeline items
4. 场景任务    你到宿舍前台，请用 check in 完成对话         ← pipeline items
5. 复练标记    答错/跳过/未使用时加入复练                  ← ExpressionItem
```

**交互流程**：

- **释义**：展示英文释义 + 中文 + 音标 + 音频（复用现有 `Vocabulary` 数据）
- **搭配**：展示 `collocations` 列表，用户跟读每个搭配
- **替换造句**：显示 `outputTraining.pipeline[].vocab_drill` 中的中文提示 → 用户说出英文 → AI 判断
- **场景任务**：显示微型 VN 片段 → 用户必须用目标词回答 → AI 判断（`targeted_output` 模式）
- **复练标记**：正确完成不自动生成复练卡，仅在以下情况生成（见改动五）
  - 答错或跳过
  - 高优先级词汇在 VN 中未使用
  - 用户主动收藏
  - AI 判断为不自然表达

### 4.5 数据获取

前端从 `practiceApi.getTopicDetail(topicId)` 获取 `TopicDetail`，其中 `topic.metadata.outputTraining` 携带 pipeline 数据。

**代码层需要确保**：

1. **后端** `english-practice.service.ts` 的 `getTopicDetail()` 在 select 中返回 `topic.metadata`：

```typescript
// getTopicDetail 的 Prisma query 中
select: {
  // ... 现有字段 ...
  metadata: true,  // 确保返回 metadata
}
```

2. **前端类型** `TopicDetail` 补充 `metadata` 字段：

```typescript
interface TopicDetail {
  topic: {
    // ... 现有字段 ...
    metadata?: {
      outputTraining?: {
        version: number
        enabled: boolean
        pipeline: OutputPipelineItem[]
      }
    } | null
  }
  // ...
}
```

### 4.6 进度状态（本地持久化）

```typescript
interface GuidedProgress {
  topicId: string
  chunkSubstitutions: Record<string, boolean>   // chunkId → 完成
  vocabDrills: Record<string, boolean>          // vocabId → 完成
  updatedAt: string
}
```

**MVP 阶段**：存储在 `localStorage` 或 `localDb`，不同步后端。

- 存储 key：`guided-progress:{topicId}`
- 用户完成当前 `PracticeSession`（进入 analysis）后清理
- 原因：移动端容易切后台、误触返回、被系统杀掉，如果不持久化会导致已完成的 guided 步骤丢失

不持久化到后端 — guided 进度是当前 session 的临时状态，不需要跨设备同步。

## 5. 改动三：AI Judge 双模式

### 5.1 问题

当前 `judgeDialogueTurn` 的 system prompt 明确说 Target chunks 是 optional，**用户可能全程用简单句通关，却没练到目标表达**。

### 5.2 方案

在 `judgeDialogueTurn` 请求中增加 `mode` 字段：

```typescript
type JudgeMode = 'communicative' | 'targeted_output'
```

**communicative（VN 主线用）**：
- 现有逻辑不变：意图清晰即通过，Chunk 非必须

**targeted_output（输出训练用）**：
- 必须使用目标 Chunk / 必须包含 essentialSlots
- 不满足标记未通过并给出提示

### 5.3 API 改动

```typescript
// POST /practice-ai/dialogue-turn
{
  // ... 现有字段 ...
  mode: 'targeted_output',             // 新增
  requiredChunks?: ["I'd like to..."],  // 新增：期望使用的 Chunk
  targetWords?: ["check in"],          // 新增：期望使用的单词（词汇训练用）
  essentialSlots?: ["new student"],    // 新增：必须包含的关键词
  allowParaphrase?: boolean,            // 新增
}
```

`targetWords` 和 `requiredChunks` 的区别：

| 字段 | 适用场景 | 示例 |
|------|---------|------|
| `requiredChunks` | Chunk 替换练习 / VN 任务 | `["I'd like to...", "Could you...?"]` |
| `targetWords` | 词汇输出练习 | `["check in", "student ID", "room key"]` |

System prompt 增加一段：

```
## Targeted Output Mode

When mode is "targeted_output", evaluate only the targets provided in the request.

- If requiredChunks are provided, the learner should use one of them or a clear equivalent structure.
- If targetWords are provided, check whether the learner naturally used those words or close inflected forms.
- If essentialSlots are provided, check whether the required meaning/content was expressed.
- Return targetWordsUsed as the target words used naturally.
- Return missingTargets as required chunks, target words, or slots that were not expressed.
- If any required target is missing, mark "passed": false unless allowParaphrase is true and the learner expressed the same meaning naturally.
```

### 5.4 使用场景

| 场景 | mode |
|------|------|
| VN 主线剧情 | `communicative`（默认） |
| Chunk 替换练习提交 | `targeted_output` |
| Vocab 替换造句 | `targeted_output` |
| 二次输出确认 | `targeted_output` |

## 6. 改动四：judgeDialogueTurn 返回 correction/retry

### 6.1 问题

当前 AI 判断后，用户表达不自然但意思通（如 "I want check in"），AI 标记 `passed: true`，**用户不需要重说**，错误被固化。

**关键区别**：你原来的 VN 逻辑已有"用户没通过 → 不推进剧情"的卡住机制，但那是**任务失败卡住**（用户没表达出任务意图）。新增的 retry 是**修正型卡住**（意思通了但表达质量不够，强制重说）。

| 对比维度 | 旧逻辑（任务失败卡住） | 新逻辑（修正型卡住） |
|---------|---------------------|--------------------|
| 卡住原因 | 用户没表达出任务意图 | 用户表达出了意思，但不自然 |
| AI 返回 | `passed: false` | `passed: true` + `retryRequired: true` |
| 训练目标 | 确保剧情能理解 | 确保用户改正表达 |
| 是否生成 correction | 不一定 | 必须返回 correction / upgraded |
| 是否记录 retry | 没有明确结构 | `isRetry` + `parentTurnId` |
| 是否进入复练 | 不一定 | 可自动生成错句/升级卡 |

### 6.2 返回字段扩展

当前返回 → 改为：

```json
{
  "intent": "ask_for_check_in",
  "passed": true,
  "objectiveCompleted": ["..."],
  "chunksUsed": ["..."],
  "inkVariables": { ... },
  "feedback": "意思能听懂，但表达不自然。",
  "confidence": 0.86,
  "correction": "I'd like to check in.",              // 新增：纠正版本
  "upgraded": "I'd like to check in, please.",         // 新增：更自然版本
  "retryRequired": true,                               // 新增：是否需要重说
  "retryPrompt": "请用 I'd like to check in, please. 再说一遍。",
  "focusChunk": "I'd like to...",                      // 新增：需要重点练的 Chunk
  "grammarIssues": [
    { "type": "grammar", "original": "I want check in", "correction": "I'd like to check in" }
  ]
}
```

### 6.3 返回字段补充：targetWordsUsed / missingTargets

在 `targeted_output` 模式下，额外返回词汇使用情况，使 Chunk 训练和词汇训练共用同一接口：

```json
{
  // ... 6.2 节基础返回字段 ...
  "targetWordsUsed": ["check in"],        // 用户实际使用的目标词
  "missingTargets": ["room key"]          // 未使用的目标词
}
```

### 6.4 System Prompt 改动

在 `english-practice-ai.service.ts` 的 `judgeDialogueTurn` system prompt 末尾增加：

```
## Correction & Retry
When the learner's response is understandable but unnatural:
1. Set "passed" to true (communication succeeded).
2. Provide a natural "correction" version.
3. Provide a more fluent "upgraded" version.
4. Set "retryRequired" to true, with a friendly "retryPrompt".
5. Identify one "focusChunk" the learner should practice.
6. List grammar issues in "grammarIssues". Be concise (1-3 items max).
```

### 6.5 前端处理

当 `retryRequired === true` 时：

1. 显示 AI 反馈卡片（含 correction / upgraded）
2. **不立即推进**到下一轮 NPC 对话
3. 显示 retryPrompt + 输入框
4. 用户再次输入 → 调用 `judgeDialogueTurn`（`mode: 'targeted_output'` + `requiredChunks: [focusChunk]`）
5. 通过后才推进 Ink 剧情

**容错**：连续 2 次重说仍不通过，允许跳过（记录到 `PracticeTurn.isRetry`）。

### 6.6 判断条件

| 用户输入 | retryRequired |

## 7. 改动五：Analysis 自动生成 ExpressionItem 复练卡片

### 7.1 问题

当前 `analyzePracticeSession` 完成后，分析结果只展示在 `PracticeAnalysisPanel` 中，**第二天不会再看到**这些错句和表达。

### 7.2 方案

提取为独立方法 `generateReviewCardsFromAnalysis`，放在 `summarizePracticeSession` 中调用，不直接把逻辑塞在 AI 处理方法里：

```typescript
// english-practice-ai.service.ts

async summarizePracticeSession(session, userId) {
  const result = await this.callAiForAnalysis(session)

  if (result.analysis) {
    await this.generateReviewCardsFromAnalysis(userId, session.id, result.analysis, session.topicId)
  }

  return result
}

// ── 独立方法 ──

private async generateReviewCardsFromAnalysis(
  userId: string,
  sessionId: string,
  analysis: any,
  topicId: string,
) {
  // 从 grammarHighlights 生成错句卡
  for (const issue of analysis.grammarHighlights ?? []) {
    if (issue.original && issue.correction) {
      await createIfNotExist(this.prisma, {
        userId, type: 'error_sentence',
        original: issue.original, corrected: issue.correction,
        sourceType: 'PracticeSession', sourceId: sessionId,
        sourceSnapshot: { topicTitle, round: issue.round },
        masteryStatus: 'learning', nextReviewAt: nextDay(),
      })
    }
  }

  // 从 chunkUsageAnalysis 生成未使用的 Chunk 卡
  for (const chunk of analysis.chunkUsageAnalysis ?? []) {
    if (!chunk.used) {
      await createIfNotExist(this.prisma, {
        userId, type: 'chunk', chunkText: chunk.chunk, original: chunk.chunk,
        sourceType: 'PracticeSession', sourceId: sessionId,
        masteryStatus: 'learning', nextReviewAt: nextDay(),
      })
    }
  }

  // 从 upgradedAnswer 生成升级卡
  if (analysis.upgradedAnswer?.natural) {
    await createIfNotExist(this.prisma, {
      userId, type: 'upgraded',
      original: analysis.upgradedAnswer.original,
      corrected: analysis.upgradedAnswer.natural,
      sourceType: 'PracticeSession', sourceId: sessionId,
      masteryStatus: 'learning', nextReviewAt: nextDay(),
    })
  }

  // 高优先级词汇未使用 → 词汇卡（见改动六）
  await this.generateVocabReviewCards(userId, sessionId, analysis, topicId)
}
```

这样调试、重跑、禁用自动生成都方便。

### 7.3 去重策略

去重用**组合条件**，不是简单的 `sourceType + sourceId`：

```typescript
async function createIfNotExist(prisma: PrismaService, data: any) {
  const where: any = {
    userId: data.userId,
    sourceType: data.sourceType,
    sourceId: data.sourceId,
    type: data.type,
  }
  // 不同卡片类型用不同的内容字段去重
  if (data.type === 'error_sentence') {
    where.original = data.original
    where.corrected = data.corrected
  } else if (data.type === 'chunk') {
    where.chunkText = data.chunkText
  } else if (data.type === 'word') {
    where.original = data.original
  } else if (data.type === 'upgraded') {
    where.original = data.original
    where.corrected = data.corrected
  }

  const existing = await prisma.expressionItem.findFirst({ where, select: { id: true } })
  if (!existing) {
    await prisma.expressionItem.create({ data })
  }
}
```

## 8. 改动六：词汇输出训练

### 8.1 问题

当前 `Vocabulary` 所有数据都是"输入向"的——用户只能看、听、理解，没有输出训练。

### 8.2 策略：两个层次

```
Vocabulary 表          → 词本身的通用知识（全局）
TrainingTopic.metadata → 这个场景怎么练这个词（局部）
```

| 层次 | 内容 | 存储位置 |
|------|------|---------|
| 通用层 | `outputPriority` + `collocations` | `Vocabulary` 表新增字段 |
| 场景层 | 中文提示、目标词、场景描述 | `TrainingTopic.metadata.outputTraining.pipeline[].vocab_drill` |

### 8.3 通用层：Vocabulary 扩展

```prisma
model Vocabulary {
  // ... 现有字段不变 ...
  outputPriority String @default("low")  // low | medium | high
  collocations   Json?  // [{ "collocation": "check in at the front desk", "zh": "在前台办理入住" }]
}
```

**不增加 `outputDrills`**。场景化的练习数据放在 `TrainingTopic.metadata` 中，避免 Vocabulary 变成一个巨大的全局 JSON。

### 8.4 场景层：TrainingTopic.metadata 中的 vocab_drill

```json
{
  "type": "vocab_drill",
  "vocabs": [
    {
      "vocabId": "xxx",
      "promptZh": "你刚到宿舍前台，想办理入住。",
      "targetWords": ["check in", "student ID", "room key"],
      "suggestedAnswer": "Hi, I'd like to check in. Here is my student ID. Could I get my room key?"
    },
    {
      "vocabId": "xxx",
      "promptZh": "我想办理入住。",
      "targetWords": ["check in"],
      "suggestedAnswer": "I'd like to check in."
    }
  ]
}
```

### 8.5 单词分类

| outputPriority | 说明 | 示例 |
|:-------------:|------|------|
| `low` | 只需看懂、听懂，不要求输出 | registration, accommodation, orientation |
| `high` | 必须会用在句子中 | check in, borrow, appointment, key, late, lost, available |

判断原则（满足任一即可标记 `high`）：
1. 场景对话中的核心动词/名词
2. 可以与 Chunk 绑定（如 appointment → make an appointment）
3. 日常生活高频词
4. 不会影响基本交流

### 8.6 词汇复练卡自动生成

在 analysis 阶段，从 `TrainingTopicVocab` 中找出 `outputPriority = 'high'` 且用户本次未使用的词，生成词汇复练卡：

```typescript
async function generateVocabReviewCards(prisma, userId, sessionId, topicId) {
  const highPriorityVocabs = await prisma.trainingTopicVocab.findMany({
    where: { topicId, vocab: { outputPriority: 'high' } },
    include: { vocab: true },
  })
  for (const tv of highPriorityVocabs) {
    const usedInTurns = await prisma.practiceTurn.findMany({
      where: { sessionId, userText: { contains: tv.vocab.word, mode: 'insensitive' } },
    })
    if (usedInTurns.length === 0) {
      await createIfNotExist(prisma, {
        userId, type: 'word', original: tv.vocab.word,
        sourceType: 'PracticeSession', sourceId: sessionId,
        masteryStatus: 'learning', nextReviewAt: nextDay(),
      })
    }
  }
}
```

### 8.7 词汇与 Chunk 的绑定

```
Vocabulary(appointment)
  → 绑定 Chunk: "make an appointment"     ← TrainingTopicChunk
    → 替换：I'd like to make an appointment.
    → 场景：诊所预约
```

数据上通过 `TrainingTopicVocab` 和 `TrainingTopicChunk` 已天然关联（同一 `TrainingTopic` 下），前端展示时交叉引用。

## 附录 A：Warmup 题型详细说明

> 本文内容原为独立文档 `warmup-pipeline-exercise-types.md`，现已合并至此。描述知识点练习流水线中的所有题型、覆盖的语言素材及输出训练需求。

### A.1 题型总览

| # | 题型 | 类型标识 | 覆盖素材 | 方向 | 核心目的 |
|---|------|---------|---------|------|---------|
| 1 | **句块/单词替换** | `chunk_substitution` | 单词 / 句块 | 中→英 / 英→中 | 在设定语境中反复使用目标表达 |
| 2 | **一词多句** | `vocab_sentence_building` | 单词 + 句块 | 中→英 / 英→中 | 一个词在多种句型框架中的灵活运用 |
| 3 | **长句拆解** | `sentence_decomposition` | 句子 | 中→英 | 从简单句逐级扩展到复杂长句 |
| 4 | **句型操练** | `pattern_drill` | 句型 | 中→英 / 英→中 | 语法框架固定 + 槽位内容可变 |

### A.2 句块/单词替换（`chunk_substitution`）

**定位**：基础输出训练。给定核心词或句块，在多条中文提示下反复输出含该词/句块的英文。

**覆盖素材**：
- `kind='word'` → 单词（如 `carefully`）
- `kind='chunk'` → 句块（如 `I'd like to...`、`check in`）

**方向**：`zh_to_en`（输出训练，默认） | `en_to_zh`（理解训练）

**数据结构**：
```json
{
  "type": "chunk_substitution",
  "kind": "word", "direction": "zh_to_en",
  "title": "用 carefully 造句",
  "chunk": "carefully", "chunkMeaning": "仔细地",
  "items": [
    { "zh": "仔细听老师说。", "answer": "Listen to the teacher carefully." },
    { "zh": "请仔细检查你的作业。", "answer": "Please check your homework carefully." }
  ]
}
```

### A.3 一词多句（`vocab_sentence_building`）

**定位**：一个核心词汇搭配多个句型框架，每个框架下多条练习。体会同一词在不同句式中的用法。

**数据结构**：
```json
{
  "type": "vocab_sentence_building",
  "vocabWord": "easily", "vocabMeaning": "容易地",
  "direction": "zh_to_en",
  "patterns": [
    {
      "chunk": "She easily...",
      "items": [
        { "zh": "她轻松解决了问题。", "answer": "She easily solved the problem." },
        { "zh": "她轻松通过了考试。", "answer": "She easily passed the exam." }
      ]
    }
  ]
}
```

> **与 chunk_substitution 的区别**：chunk_substitution（word 模式）是单一目标词多语境造句；vocab_sentence_building 是一个词 × 多个句型框架。

### A.4 长句拆解（`sentence_decomposition`）

**定位**：渐进式复杂度训练。将复杂长句拆解为 5 个递增层级（核心句 → 加对象 → 加程度 → 加频率 → 完整表达）。

**数据结构**：
```json
{
  "type": "sentence_decomposition",
  "levels": [
    { "level": 1, "label": "核心句", "en": "She speaks well.", "zh": "她说得好。" },
    { "level": 2, "label": "加对象", "en": "She speaks English well.", "zh": "她英语说得好。", "highlight": "English" },
    { "level": 5, "label": "完整表达", "en": "She always speaks English very well because she practices every day.", "zh": "她总是英语说得非常好，因为她每天练习。", "highlight": "because she practices every day" }
  ]
}
```

每一级要求用户：看上一级 → 看中文新增意图 → 输入完整新句。

### A.5 句型操练（`pattern_drill`）

**定位**：语法框架固定 + 槽位内容可变。用同一结构表达不同意思。

**数据结构**：
```json
{
  "type": "pattern_drill",
  "pattern": "I'd like to [verb]...",
  "patternMeaning": "我想要...",
  "slots": ["verb"],
  "items": [
    { "zh": "我想点一杯咖啡。", "answer": "I'd like to order a coffee.", "slotValues": { "verb": "order" } }
  ]
}
```

**AI 判断要点**：不要求完全等于参考答案，而应判断：固定结构是否成立、槽位语义是否正确、语序是否自然。如 `Could you tell me where is the library?` → `weak`（语序错）。

## 附录 B：移动端交互设计与复练机制

> 本文内容原为独立文档 `移动端知识点练习交互与复练方案.md`，现已合并至此。关注移动端 warmup dialog 的交互设计、训练回合结构和复习调度。

### B.1 现状与问题

**题型已存在，但剂量不够**：当前 warmup 更像是"做几道题验证会不会"，单个 step 一题一过，强度不足以脱口而出。

**反馈像判题不像训练**：
```
现在：中文提示 → 输入英文 → AI 判断 → 通过/再试
目标：看例句 → 半遮挡 → 中文提示输出 → 换词输出 → 无提示召回
```

**复习闭环不完整**：答错、看答案、跳过、VN 未使用→都应是"待复练资产"，但当前没有自动沉淀。

### B.2 一句话训练回合

每步改为 4 层：

```
1. Prime    看例句/看结构
2. Build    有提示输出
3. Recall   无答案输出
4. Speed    快速再来一次
```

**通过标准（四档）**：

| 状态 | 含义 | 后续 |
|------|------|------|
| `strong` | 无提示、自然、命中目标 | 当前 session 不再出现 |
| `ok` | 基本正确，但慢/不够自然 | 本轮稍后再出现 1 次 |
| `weak` | 看提示后才对，或需要 correction | 加入今日复练 |
| `miss` | 错误/跳过/没输出出来 | 加入今日复练 + 次日复练 |

### B.3 四题型升级设计

**句块替换 → 句块连击**：每个核心 chunk 至少 3-5 次替换 + 1 次微场景 + 1 次无提示召回，不是 1 题而是 5-7 个短动作。

**一词多句 → 词汇训练栈**：`appointment` → `make/have/cancel/reschedule an appointment` + 微场景。复练粒度是 `word + collocation + example`，不是孤立的词。

**句子拆解 → 渐进复述**：每级必须做一次输出（看上级 → 看新增 → 输入完整新句），最终 Recall 合上答案再写一遍。

**句型操练 → 槽位变速训练**：突出"固定骨架 + 可变槽位"，AI 判断应宽松（结构对 + 槽位对 = 通过），语序错判 weak。

### B.4 提示分三层

| 层级 | 内容 | 影响评分 |
|------|------|----------|
| Hint 1 | 给目标 chunk / pattern | 仍可 strong |
| Hint 2 | 给首词 / 词序 | 最多 ok |
| Hint 3 | 显示完整答案 | 标记 weak，需要复练 |

### B.5 复习机制

**Warmup 内部复现**：同一 session 内做小间隔（第 1 题 A，第 5 题 A 换语境，第 10 题无提示召回）。

**今日复练**：weak/miss 在结束前再练一次，通过才进 VN。

**次日复练**：生成 ExpressionItem 复练卡（保存 prompt + 答案 + 错因），调度节奏如下：
```
miss  → 10 分钟后 / 今天结束前 / 明天
weak  → 今天结束前 / 明天
ok    → 3 天后
strong→ 7 天后或不生成
```

### B.6 推荐的数据结构扩展

```typescript
type WarmupTrainingMeta = {
  minReps?: number
  masteryTarget?: 'recognize' | 'produce' | 'automatic'
  inputMode?: 'typing_first' | 'speech_first'
  reviewPolicy?: 'none' | 'on_fail' | 'on_weak' | 'always'
  timeLimitSec?: number
}

type WarmupAttempt = {
  stepId: string; itemType: string
  userAnswer: string; normalizedAnswer?: string
  score: 'strong' | 'ok' | 'weak' | 'miss'
  usedHintLevel: 0 | 1 | 2 | 3
  latencyMs?: number; retryCount: number
  targetUsed: boolean; correction?: string
  createdAt: string
}
```

### B.7 建议的 MVP 版本

```
12 题训练队列：
  4 题句块替换 + 3 题一词多句 + 3 题句型操练 + 2 题句子拆解/复述

每题：中文提示 → 用户输出 → AI 判断 → strong/ok/weak/miss
结束前：weak/miss 再练一次
结束后：weak/miss 生成复练卡
```
