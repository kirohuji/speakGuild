# App Store 截图指南

## 尺寸要求

| 设备 | 竖屏 | 横屏 |
|---|---|---|
| iPhone 6.5"（iPhone XS Max / 11 Pro Max / 11 / 12 / 13 / 14 / 15 / 16） | 1242 × 2688 px | 2688 × 1242 px |
| iPhone 6.7"（iPhone 14 Pro Max / 15 / 16 Pro Max） | 1284 × 2778 px | 2778 × 1284 px |

> 建议只传 **6.7" 尺寸**（1284 × 2778 / 2778 × 1284），Apple 会自动缩放适配其他机型。

## 截图数量

- **App 预览（视频）**：最多 3 个（30 秒内）
- **截图**：最多 10 张
- 建议上传 **6-8 张竖屏 + 2 张横屏**

## 推荐截图页面

### 竖屏（建议 6 张）

| # | 页面 | 路由 | 说明 |
|---|---|---|---|
| 1 | **首页** | `/` | 时钟 + 每日名言 + 签到入口，展示 App 氛围 |
| 2 | **剧本模式** | `/script` | 互动剧本列表，展示场景化学习 |
| 3 | **探索模式** | `/explore` | 地图 + 地点探索，展示自由对话特色 |
| 4 | **练习记录** | `/learning` | 学习计划/练习记录 |
| 5 | **个人中心** | `/profile` | 学习统计概览 |
| 6 | **表达库** | `/expressions` | 生词本 / 句块库 |

### 横屏（建议 2 张）

| # | 页面 | 路由 |
|---|---|---|
| 1 | 首页 | `/` |
| 2 | 探索模式 | `/explore` |

## 如何截图

### 方案 A：Xcode Simulator（推荐）

```bash
# 1. 启动后端
cd apps/backend && pnpm dev

# 2. 新终端，启动前端
cd apps/frontend && pnpm dev

# 3. 新终端，启动 iOS 模拟器
cd apps/frontend && npx cap run ios
```

模拟器启动后：

1. 用测试账号登录：`user@engjourney.local` / `user123456`
2. 导航到各页面
3. `⌘Cmd + S` 截图（保存到桌面）
4. 用 Preview 打开检查尺寸是否符合

### 方案 B：真机

1. 通过 Xcode 构建到真机
2. 登录测试账号
3. 导航到各页面
4. **实体按钮**：电源键 + 音量上键
5. 截图会自动保存到「照片」App

## 上传前检查清单

- [ ] 截图尺寸是否正确（1284 × 2778 或 1242 × 2688）
- [ ] 截图中不包含模拟器边框、地址栏、状态栏（模拟器截图自带状态栏，App Store 认可）
- [ ] 图片为 PNG 格式，无压缩失真
- [ ] 第一张截图最重要，选最能体现 App 核心价值的页面
- [ ] 截图文案不要包含占位文本（如 "暂无数据"）

## 注意事项

- 截图中的**文字必须使用简体中文**（App Store 中国区要求）
- 如果英文水平较高，可以准备一套英文截图用于海外上架
- 截图不要包含未公开功能或测试数据
- iPad 截图（如果需要）尺寸：2048 × 2732 px（12.9"）或 1620 × 2160 px（11"）
